Overview & Core Concepts

What is Text Embedding? Converting text into numerical vectors for machine learning models. Different methods create different representations with unique strengths.

Purpose: Ensure reproducible data loading across different environments with graceful degradation.

  • Priority 1 - Local CSV: Reproducible, no dependencies, fastest (0-5 seconds)
  • Priority 2 - Kaggle API: Requires setup, 50K+ reviews, ~30-60 seconds
  • Priority 3 - Hugging Face: Public datasets, no credentials, auto-download
  • Priority 4 - Synthetic Data: Fallback for demo/testing, fast generation
  • Configuration: RUN_QUICK flag (smaller subset for fast iteration), MAX_ROWS limit
try: data = pd.read_csv('local_data.csv') except FileNotFoundError: try: data = kaggle_download() except: try: data = load_from_huggingface() except: data = generate_synthetic_data()
Best Practice: Always log which data source was loaded for reproducibility and debugging!

Purpose: Convert raw text into tokens for embedding methods to process.

  • Type Conversion: Ensure all text is string type (handle NaN/None)
  • Basic Tokenization: Split by whitespace: "hello world" → ["hello", "world"]
  • Vocabulary Building: Collect unique tokens, create word-to-index mapping
  • OOV Strategy: Unseen words in test set handled by method (Word2Vec averages, DistilBERT has subwords)
  • Document Representation: Aggregate token embeddings to document level
text = "This is a sample" tokens = text.lower().split() # ["this", "is", "a", "sample"] vocab = {word: idx for idx, word in enumerate(set(all_tokens))} token_ids = [vocab[t] for t in tokens if t in vocab]
Key Insight: Vocabulary size heavily impacts embedding dimension for shallow methods (TF-IDF, BoW) but not dense methods (Word2Vec, BERT).

Purpose: Fast, interpretable baseline methods using frequency statistics.

  • One-Hot / BoW: Binary or count presence of words. Example: [1, 0, 2, 1] for [word1, word2, word3, word4]
  • TF-IDF Formula: TF(t,d) = count(t in d) / total_words(d); IDF(t) = log(N / count(d with t))
  • Count N-grams: Captures word sequences: "good movie" as bigram weights order

Characteristics:

  • ✅ Pros: Lightning fast (milliseconds), highly interpretable, no training required, works for large vocabularies
  • ❌ Cons: Sparse high-dimensional vectors, no semantic meaning ("good" ≠ "great"), ignores word order (BoW)
  • 📊 Typical Dimension: 5K-100K depending on vocabulary size
from sklearn.feature_extraction.text import TfidfVectorizer vectorizer = TfidfVectorizer(max_features=1000) X = vectorizer.fit_transform(texts) # Output shape: (n_samples, 1000) - sparse matrix

Purpose: Neural embeddings capturing semantic relationships in low-dimensional space.

  • Word2Vec (2013): Skip-gram or CBOW architecture, learns from word contexts. Analogy: "king" - "man" + "woman" ≈ "queen"
  • FastText (2017): Extends Word2Vec with subword information, handles misspellings and rare words
  • GloVe (2014): Pre-trained on Wikipedia (6B tokens) or Common Crawl (840B), immediate use without training

Document Representation: Average word vectors: doc_emb = mean([emb(word) for word in doc])

Characteristics:

  • ✅ Pros: Dense (low-dim ~300), semantic relationships, fast inference, transferable to new tasks
  • ❌ Cons: Same vector per word (no context), averaging loses word order, requires pre-training or corpus
  • 📊 Typical Dimension: 100-300, much smaller than TF-IDF
from gensim.models import Word2Vec model = Word2Vec(sentences, vector_size=300, window=5, min_count=2) emb = model.wv['word'] # Get word embedding # doc_embedding = np.mean([model.wv[w] for w in doc_words], axis=0)

Purpose: Context-aware embeddings using transformer architecture for state-of-the-art semantic understanding.

  • BERT (2018): 12-24 layers, bidirectional attention, best semantic quality but slow
  • DistilBERT (2019): 6 layers, 40% faster, 60% smaller, 97% of BERT quality - recommended for production
  • Context-Awareness: Same word ("bank") gets different vectors depending on context ("river bank" vs "savings bank")
  • Pre-training: Trained on Wikipedia (12GB) + Books (3.6GB) corpus with masked language modeling

Document Representation: Mean pooling of all token embeddings: doc_emb = mean(all_token_embeddings)

Characteristics:

  • ✅ Pros: Context-aware, superior semantic quality, handles complex language, state-of-the-art accuracy
  • ❌ Cons: Computationally expensive (0.5-2s per doc), requires GPU for speed, large model size (268MB)
  • 📊 Typical Dimension: 768 (DistilBERT), requires GPU for real-time applications
from sentence_transformers import SentenceTransformer model = SentenceTransformer('distilbert-base-uncased') embedding = model.encode('This is a sample text') # Shape: (768,), cached in GPU memory for batch processing

Purpose: Fair evaluation where different embeddings are compared on identical data, controlling for data randomness.

  • Split Ratio: 80/20 (training/test) or 70/30 depending on data size
  • Stratification: Preserve class distribution: if 60% positive, both splits have 60% positive
  • Random Seed: SEED=42 (arbitrary but widely used) for reproducibility across runs
  • Single Split: Use SAME train/test split for all 7+ embedding methods
  • Rationale: Isolates embedding quality; eliminates data randomness as confounding variable
from sklearn.model_selection import train_test_split X_train, X_test, y_train, y_test = train_test_split( X, y, test_size=0.2, stratify=y, random_state=42 ) # Use SAME split for all embedding methods for method in all_methods: X_train_emb = embed(X_train, method) X_test_emb = embed(X_test, method) evaluate(X_train_emb, X_test_emb)
Critical: Do NOT resplit data for each embedding method. This would introduce data randomness and make comparison unfair.

Purpose: Measure embedding quality through downstream task performance (classification accuracy).

  • Classifier: Logistic Regression (simple, interpretable, not method-specific)
  • Metrics: Accuracy (overall), Precision (true positives), Recall (coverage), F1 (balance)
  • Setup: Same classifier, same hyperparameters (C=1.0, solver='lbfgs'), same train/test data
  • Evaluation: Train on embedding vectors, test accuracy indicates embedding quality

Typical Results (Sentiment Classification):

  • TF-IDF: 89-91% accuracy (baseline)
  • Word2Vec/FastText: 91-93% accuracy
  • GloVe: 92-94% accuracy
  • DistilBERT: 95-97% accuracy (best)
from sklearn.linear_model import LogisticRegression from sklearn.metrics import accuracy_score, f1_score clf = LogisticRegression(random_state=42, max_iter=1000) clf.fit(X_train_emb, y_train) preds = clf.predict(X_test_emb) acc = accuracy_score(y_test, preds) f1 = f1_score(y_test, preds) print(f"{method}: Accuracy={acc:.3f}, F1={f1:.3f}")
Key Insight: Higher quality embeddings → Higher classifier accuracy. This is your primary evaluation metric.

Selection Criteria Decision Tree
Do you have < 1 second latency requirement?

→ YES: Use TF-IDF (instant), Word2Vec (fast)
→ NO: Can use DistilBERT (0.5-2s) or BERT (2-5s)

Do you need interpretability?

→ YES: Use TF-IDF (see which words matter)
→ NO: Use DistilBERT (black-box but powerful)

Do you have pre-trained models available?

→ YES: Use GloVe or DistilBERT (immediate use)
→ NO: Train Word2Vec on your domain corpus

Is context crucial for your task?

→ YES: Use DistilBERT or BERT (handles ambiguity)
→ NO: Word2Vec or GloVe sufficient

Pro Tips & Best Practices

Decision Matrix Based on Your Requirements:

  • Speed Critical (< 100ms): One-Hot, TF-IDF, Count N-grams (sklearn)
  • Semantic Quality Important: Word2Vec, FastText, GloVe, DistilBERT
  • No Training Time: GloVe (pre-trained), TF-IDF (no training), DistilBERT (pre-trained)
  • Custom Domain Data: Train Word2Vec on your corpus (FastText better for rare words)
  • Production System: GloVe (reliable), or DistilBERT (best quality)
  • Research/Competition: DistilBERT, BERT, fine-tuned transformers
  • Resource-Constrained: TF-IDF (minimal memory), DistilBERT (smaller than BERT)
  • GPU Available: Transformers (DistilBERT 0.5s with GPU, 5s without)
Pro Tip: Start with TF-IDF (fast baseline). If accuracy insufficient, try Word2Vec. If still needed, upgrade to DistilBERT.

Essential Steps for Valid Comparison:

  • 1. Single Data Split: Load data once, split into train/test (80/20, stratified, SEED=42), use for ALL embeddings
  • 2. Same Classifier: Use LogisticRegression (or any single choice) with SAME hyperparameters for all methods
  • 3. Same Metrics: Measure Accuracy, Precision, Recall, F1-score for each embedding method
  • 4. Multiple Runs: Run 3-5 times with different seeds (42, 123, 456, 789, 999) and report mean ± std
  • 5. Document Everything: Log sklearn version, random seeds, embedding dimensions, training time, hyperparameters
  • 6. Statistical Testing: Use paired t-test to check if accuracy differences are statistically significant
results = {} for method in all_methods: accuracies = [] for seed in [42, 123, 456, 789, 999]: X_train_emb = embed(X_train_texts, method) X_test_emb = embed(X_test_texts, method) clf = LogisticRegression(random_state=seed, max_iter=1000) clf.fit(X_train_emb, y_train) acc = clf.score(X_test_emb, y_test) accuracies.append(acc) results[method] = {'mean': np.mean(accuracies), 'std': np.std(accuracies)}

Recommended Workflow for Data Management:

  • Format: .npz (numpy compressed) for fast I/O and small file sizes (~50% savings vs .npy)
  • Naming Convention: embeddings_{method}_{date}.npz (e.g., embeddings_tfidf_2024-04-25.npz)
  • Metadata File: Create JSON with shape, method, timestamp, sklearn version, layer info
  • Save Train & Test Separately: embeddings_tfidf_train.npz, embeddings_tfidf_test.npz
  • Version Control: Track hash (MD5) of files to detect accidental modifications
  • Documentation: Include README explaining each embedding file, generation parameters, and results achieved
import numpy as np import json # Save training embeddings np.savez_compressed(f'embeddings_tfidf_train.npz', X=X_train_tfidf) np.savez_compressed(f'embeddings_tfidf_test.npz', X=X_test_tfidf) # Save metadata metadata = { 'method': 'tfidf', 'train_shape': X_train_tfidf.shape, 'test_shape': X_test_tfidf.shape, 'timestamp': '2024-04-25', 'sklearn_version': '1.3.0', 'vectorizer_params': {'max_features': 1000, 'lowercase': True} } with open('embeddings_metadata.json', 'w') as f: json.dump(metadata, f, indent=2) # Load later X_train_loaded = np.load('embeddings_tfidf_train.npz')['X']

Performance Optimization Techniques by Method:

  • Shallow Methods (TF-IDF): Use sparse matrices, max_features limit, reduce vocabulary, parallel processing available
  • Word2Vec/FastText: Use workers parameter (multiprocessing), increase window size, min_count threshold reduces OOV
  • Transformer (DistilBERT): Use batch_size >= 32, enable GPU (CUDA), cache embeddings after first run, reduce sequence length if possible
  • Global Optimization: RUN_QUICK=True for dev (smaller dataset, e.g., 10K samples), MAX_ROWS parameter, use fp16 precision for transformers
  • Cache Strategy: Pre-compute embeddings once, save to disk, reuse for different classifiers
import torch from sentence_transformers import SentenceTransformer # GPU optimization device = 'cuda' if torch.cuda.is_available() else 'cpu' model = SentenceTransformer('distilbert-base-uncased') model = model.to(device) # Batch processing for speed embeddings = model.encode(texts, batch_size=32, show_progress_bar=True) # With GPU: ~100 texts/second; Without GPU: ~10 texts/second

Debugging Guide for Common Errors:

  • OOV Words (Out-of-Vocabulary): Use FastText (handles subwords), lower min_count in Word2Vec, or include special [UNK] token. DistilBERT has 30K token vocabulary
  • Memory Error (MemoryError/CUDA OOM): Reduce MAX_ROWS, enable sparse matrices for TF-IDF, reduce batch size for transformers, use fp16 precision, delete intermediate arrays explicitly
  • Slow Transformer Inference: Use DistilBERT (6 layers) instead of BERT (12), increase batch_size to 64, enable GPU acceleration, cache embeddings, reduce sequence length (truncate to 128 tokens)
  • Download Failures: Check internet, verify credentials (Kaggle), clear cache (~/.cache/huggingface), use fallback mechanism, manually download and use local files
  • Non-Reproducible Results: Set RANDOM_SEED in Python, set TensorFlow/PyTorch seeds, fix numpy.random.seed(42), disable hardware acceleration randomness (CUDA_LAUNCH_BLOCKING=1)
  • Low Accuracy Across All Methods: Check data quality (NaN values), verify labels (y_train distribution), ensure proper preprocessing, confirm train/test split working

Production Deployment Checklist:

  • Modular Functions: Create embed_text(text, method) function with error handling, logging, fallbacks
  • Configuration: Store hyperparameters in config.yaml (vocabulary size, model path, batch size, timeout)
  • Logging & Monitoring: Log method used, embedding time, errors, track service latency, monitor model size in memory
  • Versioning: Version embeddings (v1.0, v1.1), track model updates, maintain backward compatibility with old embeddings
  • Model Serving: Use FastAPI or Flask for REST API, containerize with Docker, scale with Kubernetes/serverless
  • Caching: Cache embeddings for repeated texts, use Redis/memcached, TTL strategy for freshness
  • A/B Testing: Run experiment: 50% users get TF-IDF, 50% get DistilBERT, measure impact on key metrics (accuracy, latency, cost)
  • CI/CD Pipeline: Automated tests for accuracy, latency benchmarks, data drift detection, model updates triggered by performance drops
