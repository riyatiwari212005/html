GPT-Style Transformers: Text Generation

🎯 What You'll Learn Decoder-only models that predict the next token based on previous tokens. These power ChatGPT, text completion, and creative writing applications!

📚 Core Concepts

🔄 Autoregressive Generation

Predict next token based on previous ones. One token at a time, like typing with a crystal ball predicting what you'll type next!

🔒 Causal Masking

Future tokens cannot influence past predictions. This creates the left-to-right sequential pattern essential for generation.

💾 Memory Optimization

FP16 precision, gradient checkpointing, and batch optimization to fit models on consumer GPUs.

🏗️ Models Covered

Model Parameters Memory Best For
GPT-2 Small 124M ~2.5GB Learning, text generation
GPT-Neo 125M 125M ~3GB Open-source GPT, fine-tuning

🔒 Understanding Causal Masking

The Problem: Without causal masking, a model could "cheat" by looking at future tokens during training.

💡 Visual Example (4-token sequence)
Position 0 ("The"): Can attend to [0] ✓
Position 1 ("cat"): Can attend to [0, 1] ✓
Position 2 ("sat"): Can attend to [0, 1, 2] ✓
Position 3 ("down"): Can attend to [0, 1, 2, 3] ✓
Attention Mask Matrix:
Lower triangular pattern (1 = can attend, 0 = cannot):

[1 0 0 0]
[1 1 0 0]
[1 1 1 0]
[1 1 1 1]

🎲 Text Generation Strategies

Strategy How It Works Quality Diversity Best Use
Greedy Pick highest probability token Medium None (repetitive) Fastest inference
Top-K (K=40) Sample from top 40 tokens High Good Balanced generation
Nucleus (Top-p) Sample from top tokens until cumulative prob reaches p Very High Very Good Creative writing

Temperature Control (Key Insight!):

  • Low Temperature (0.1-0.5): Very focused, deterministic output. Good for factual content, code generation
  • Medium Temperature (0.7-1.0): Balanced creativity and coherence. Default for most tasks
  • High Temperature (1.2-2.0): More random and creative. For poetry, storytelling, brainstorming
💡 Temperature Effect: How It Works
Temperature scales logits before softmax: softmax(logits / T)
• T = 0.1: Sharp distribution, high probability on top token
• T = 1.0: Original distribution
• T = 2.0: Flat distribution, all tokens equally likely

Seq2Seq Transformers: Text Transformation

🎯 What You'll Learn Encoder-Decoder architecture for translation, summarization, and text transformation. The encoder understands the input, the decoder generates the output!

🏗️ Encoder-Decoder Architecture

Unlike GPT-style models with only a decoder, Seq2Seq has two main parts:

  • Encoder: Bidirectional processing of input. Each token sees all other tokens. Creates rich context representations.
  • Decoder: Autoregressive generation with causal masking. But also has cross-attention to encoder!
  • Cross-Attention: Decoder queries ask "which encoder tokens are relevant to me?" This is how translation works!
Cross-Attention Formula:

Q (Query) = Decoder representations
K (Key) = Encoder representations
V (Value) = Encoder representations

Attention = softmax(Q·K^T / √d_k) · V

📝 T5: Text-to-Text Transfer Transformer

Key Innovation: Everything is text-to-text. Classification? Summarization? Translation? Same architecture!

🎯 T5 Task Examples
• translate English to French: Hello → Bonjour
• summarize: [article text] → [summary]
• question: What? context: [text] → [answer]
• classify: [text] → [label]

🎨 BART: Bidirectional Encoder + Autoregressive Decoder

Key Innovation: Combines BERT-style bidirectional encoding with GPT-style autoregressive decoding using denoising autoencoder pretraining.

Pretraining Strategy - Denoising Autoencoder:

  • Corrupt text using multiple strategies (masking, deletion, permutation)
  • Train encoder to process corrupted input
  • Train decoder to reconstruct original text
  • Result: Robust representations for generation tasks
🔧 BART Corruption Strategies
• Token Masking: Replace tokens with [MASK]
• Token Deletion: Remove tokens entirely
• Text Infilling: Replace spans with single mask
• Sentence Permutation: Shuffle sentence order
• Document Rotation: Rotate from random position

Why Different Corruption Types?

  • Masking: Learn to infer masked tokens from context
  • Deletion: Model must infer missing information
  • Infilling: Predict variable-length spans
  • Permutation: Learn document structure and coherence

📊 T5 vs BART: Detailed Comparison

💡 When to Use Which?
Use T5 when: Multiple diverse tasks, classification important, want unified paradigm
Use BART when: Summarization priority, generation quality paramount, bidirectional encoder benefits needed

🔗 Cross-Attention: The Bridge Between Encoder & Decoder

Cross-attention is the key mechanism! It allows the decoder to "look at" the encoder output.

Cross-Attention Formula:

Q (Query) = Decoder token representations
K (Key) = Encoder token representations
V (Value) = Encoder token representations

Output = softmax(Q·K^T / √d_k) · V

How It Works in Translation:

  • Decoder generates French token "chat" (cat)
  • Cross-attention queries: "Which source words are relevant?"
  • High attention weights on English word "cat"
  • Retrieves features about cats
  • Generates appropriate French translation
💡 Why Multi-Head Cross-Attention?
Different heads focus on different alignment patterns

💻 Code Example: Summarization with T5

Python - Summarize Text
from transformers import T5Tokenizer, T5ForConditionalGeneration # Load T5-small tokenizer = T5Tokenizer.from_pretrained("t5-small") model = T5ForConditionalGeneration.from_pretrained("t5-small") # Example text to summarize text = """ Machine learning is rapidly transforming industries. From healthcare to finance, AI systems are making decisions that were previously only possible for humans. """ # Encode with "summarize:" prefix (T5 task prefix) input_ids = tokenizer.encode( "summarize: " + text, return_tensors="pt", max_length=512, truncation=True ) # Generate summary summary_ids = model.generate(input_ids, max_length=100, num_beams=4) summary = tokenizer.decode(summary_ids[0], skip_special_tokens=True) print(f"Summary: {summary}")

Vision Transformers: Image Understanding

🎯 What You'll Learn How to apply transformer architecture to images by splitting them into patches. No CNNs needed!

🧩 Patch Embeddings: From Images to Sequences

Key Insight: Transformers work on sequences. Images are 2D. Solution? Divide image into patches and flatten them!

🔲 Patch Process
Input image: 224×224 pixels
Patch size: 16×16 pixels
Number of patches: 14×14 = 196 patches
Patch dimension: 16×16×3 = 768 values
Projected to: 768-dimensional embedding

Why patches?

  • Reduces sequence length (196 tokens vs 50,176 pixels)
  • Maintains spatial information through positioning
  • Enables processing of larger images
  • More efficient than pixel-level processing

💻 Code Example: Image Classification with ViT

Python - Classify Image with ViT
from transformers import ViTImageProcessor, ViTForImageClassification from PIL import Image import requests # Load model and processor processor = ViTImageProcessor.from_pretrained("google/vit-base-patch16-224") model = ViTForImageClassification.from_pretrained("google/vit-base-patch16-224") # Load image image_url = "https://example.com/image.jpg" image = Image.open(requests.get(image_url, stream=True).raw) # Preprocess inputs = processor(images=image, return_tensors="pt") # Forward pass outputs = model(**inputs) logits = outputs.logits # Get prediction predicted_class_idx = logits.argmax(-1).item() predicted_class = model.config.id2label[predicted_class_idx] print(f"Predicted class: {predicted_class}")

🪟 Swin Transformer: Hierarchical Attention

Key Innovation: Window-based attention that shifts between layers, enabling efficient hierarchical feature learning!

  • Window Attention: Attention computed within local windows (7×7 or 14×14)
  • Shifted Windows: Windows shift by half their size between layers
  • Hierarchical Blocks: Creates pyramid of features like CNNs
  • O(n) Complexity: Linear complexity through window-based attention
  • Multi-Scale: Natural multi-scale feature representation
💡 Window Shifting Pattern
Layer 1: Normal windows [0-7, 8-15, 16-23, ...]
Layer 2: Shifted windows [-3-4, 5-12, 13-20, ...]
Benefit: Information flows between regions!
Aspect ViT-Base Swin-Base
Attention Type Global (Full image) Local (Windows)
Complexity O(n²) O(n)
Feature Hierarchy Flat tokens 4-level pyramid
ImageNet Top-1 88.6% 90.3%

Multimodal Transformers: Text + Image + Audio + Video NEW

🎯 What You'll Learn Combine complementary signals from multiple modalities (text, image, audio, video). Build systems for text-image retrieval, captioning, VQA, and speech recognition!

🧠 Core Building Blocks

  • Modality Encoders: Turn raw inputs into embeddings (Text: tokenization → BERT, Image: patches → ViT, Audio: spectrograms → Whisper)
  • Alignment & Fusion: Combine representations across modalities (Joint embedding space like CLIP, Cross-attention like BLIP)
  • Positional/Temporal Encoding: Spatial information for images, temporal for audio/video sequences

🤖 Multimodal Models

Model Modalities Architecture Best For
CLIP Text + Image Dual encoders (contrastive) Text-image retrieval, zero-shot classification
BLIP Text + Image Encoder-decoder with cross-attention Image captioning, visual question answering
ViLT Text + Image Unified transformer (minimal visual backbone) Efficient VQA with minimal parameters
Whisper Audio + Text Encoder-decoder (speech-to-text) Automatic speech recognition (ASR), transcription
TimeSformer Video frames Spatio-temporal attention Video action recognition, classification

🔗 Understanding Multimodal Attention

Self-Attention: Within a single modality (e.g., image patches attending to each other)

Cross-Attention: Between modalities (e.g., text tokens querying image features)

Co-Attention: Bidirectional attention between two modalities simultaneously

💡 How Cross-Attention Works in BLIP (Image Captioning)
Text Token: "cat"
↓ (Query)
Attention Map: High weights on cat region patches
↓ (Retrieve)
Visual Features: Cat appearance, texture, pose

📊 Key Pipelines

🔍 Retrieval (CLIP)

Encode image & text → joint space. Similarity (cosine) → ranking, Recall@K metrics.

📝 Captioning (BLIP)

Encode image → decoder generates caption tokens. Evaluate with BLEU/CIDEr/SPICE.

❓ VQA (ViLT)

Encode (image, question) → classify answer. Accuracy on VQA datasets.

🎙️ ASR (Whisper)

Audio → features → decoder predicts text. Word Error Rate (WER) metric.

💻 Code Example: CLIP Text-Image Retrieval

Python - CLIP Image-Text Matching
import torch from transformers import CLIPProcessor, CLIPModel from PIL import Image import requests # Load CLIP model model = CLIPModel.from_pretrained("openai/clip-vit-base-patch32") processor = CLIPProcessor.from_pretrained("openai/clip-vit-base-patch32") # Load image image_url = "http://images.cocodataset.org/val2017/000000039769.jpg" image = Image.open(requests.get(image_url, stream=True).raw) # Text candidates texts = ["a photo of a cat", "a photo of a dog", "a photo of a car"] # Process inputs inputs = processor(text=texts, images=image, return_tensors="pt", padding=True) # Forward pass outputs = model(**inputs) logits_per_image = outputs.logits_per_image probs = logits_per_image.softmax(dim=1) # normalize # Display probabilities for i, text in enumerate(texts): print(f"{text}: {probs[0][i].item():.2%}")

Student model learns from larger teacher model. Compresses knowledge into smaller model.

🏗️ DistilBERT: Your First Transformer

DistilBERT is BERT made 60% smaller while retaining 97% of performance! Perfect starting point.

  • 6 transformer layers (vs 12 in BERT-base)
  • 66M parameters (vs 110M in BERT-base)
  • Knowledge distillation from BERT teacher
  • Bidirectional self-attention mechanism

🧬 Knowledge Distillation: Core Technique

How a large teacher model teaches a small student model!

  • Teacher Model: Large, accurate BERT-base (110M parameters)
  • Student Model: Small, fast DistilBERT (66M parameters)
  • Knowledge Transfer: Student learns both from task labels AND teacher outputs
  • Soft Targets: Temperature-scaled probability distributions from teacher
  • Result: 60% faster, 40% smaller, 97% performance retained!
💡 Why Soft Targets Matter
Hard target: [0, 1, 0, 0] (one-hot encoded answer)
Soft target: [0.01, 0.92, 0.05, 0.02] (reveals teacher confidence)
Student learns: Not just correct answer, but WHY it's correct!
Distillation Loss Combination:

Total Loss = α * Task Loss + (1-α) * Distillation Loss

Where:
• Task Loss: Standard classification loss
• Distillation Loss: KL divergence from teacher outputs
• α: Balance parameter (typically 0.3-0.7)

🤖 TinyBERT: Multi-Layer Distillation

TinyBERT goes beyond DistilBERT with two-stage learning:

  • Stage 1: General Distillation - Distill from pre-trained BERT on unlabeled data
  • Stage 2: Task-specific Fine-tuning - Fine-tune on downstream task with task-specific data
  • Attention Transfer: Learn to copy teacher's attention patterns
  • Hidden State Matching: Align internal layer representations
  • Only 14-22M parameters with excellent performance!
Aspect DistilBERT TinyBERT
Parameters 66M 14-22M
Layers 6 4
Training Stages Single Two (general + task-specific)
Attention Transfer No Yes (key innovation)
Speed vs BERT 2.7× 9-15×

✨ MiniLM: Attention Distillation Master

MiniLM focuses specifically on distilling attention mechanisms for excellent embeddings!

  • Deep Self-Attention Distillation: Focus on attention transfer across all layers
  • Last-Layer Attention Transfer: Preserve final attention patterns
  • Value-Relation Preservation: Maintain semantic relationships
  • Excellent for Embeddings: Outstanding sentence representations for semantic search
  • 33M parameters with state-of-the-art embedding quality
🎯 When to Use Which?
DistilBERT: General NLP tasks, text classification, all-around good
TinyBERT: Maximum compression, mobile deployment, extreme efficiency
MiniLM: Semantic search, similarity matching, embedding quality crucial

ALBERT achieves massive parameter reduction through clever parameter sharing!

  • Cross-layer Parameter Sharing: Same weights across transformer layers
  • Factorized Embeddings: Separates vocabulary size from hidden size
  • Only 12M parameters! 90% smaller than BERT-base
  • Sentence Order Prediction: Additional training task for robustness

Longformer: Long Document Processing NEW

🎯 What You'll Learn Process documents up to 4096 tokens efficiently! Using sparse attention patterns (O(n) instead of O(n²)), handle research papers, legal documents, books!

🔍 The Attention Problem

Standard BERT: O(n²) complexity for sequence length n

  • 512 tokens = 262K attention operations
  • 4096 tokens = 16.8M operations ❌ (explodes memory)
  • 16384 tokens = 268M operations ❌ (impossible!)

Real-world documents: Legal contracts (10K+ tokens), Books (100K+ tokens), Code files (30K+ lines)

⚡ Longformer's Sparse Attention Solution

Three types of attention patterns:

🔄 Sliding Window Attention

Each token attends to w/2 tokens on each side (local context). O(n·w) complexity.

🌍 Global Attention

Special tokens (CLS, questions) attend to ALL positions. Captures document-wide information.

🎯 Dilated Sliding Window

Some heads use larger gaps between attended positions. Captures long-range dependencies.

Result: O(n) Complexity!

• BERT (512 tokens): O(n²) = 262K ops
• Longformer (4096 tokens): O(n·w) = ~2M ops (13× fewer!)
• Longformer (16384 tokens): ~8M ops (still manageable!)
• Handles 8× longer sequences with 4× the operations

💾 Memory Optimization for Long Documents

Practical tips for working with long sequences:

  • Start small: Downsize documents (truncate to 2048 tokens initially)
  • Use float16: Automatic mixed precision reduces memory by 50%
  • Batch size wisely: Start with batch_size=1, increase gradually
  • Gradient accumulation: Simulate larger batches without memory overhead
  • Cache preprocessors: Don't re-tokenize documents repeatedly
  • Monitor memory: Use torch.cuda.memory_allocated() to track usage
🔧 Memory Usage Estimate
For sequence length n = 4096 tokens:
• Attention states: ~64MB
• Hidden states: ~128MB
• Gradients: ~256MB
• Total: ~450MB (vs 2+GB for BERT on same length!)

🏗️ Longformer Architecture

Combined Attention Strategy:

Input: Long document (up to 4096 tokens)
↓
Local Attention: Each token looks at nearby tokens
↓
Global Attention: Special tokens see entire document
↓
Dilated Attention: Some heads skip positions for long-range deps
↓
Output: Rich representations with O(n) complexity

Key Features:

  • Pre-trained on long documents
  • Efficient inference (2-4× faster than BERT on long sequences)
  • Maintains bidirectional understanding
  • Works with existing transformer fine-tuning approaches

🔄 Attention Patterns Visualization

BERT Full Attention: Every token attends to every other token. Quadratic pattern.

Longformer Sparse Attention: Local window + global attention points. Linear pattern!

💡 Attention Pattern Example
Position 0 (CLS): Attends to ALL positions [0-31] (global)
Position 1: Attends to [0-3] (window)
Position 2: Attends to [0-4] (window)
Position 16: Attends to [14-18] (window) + [0] (global)
