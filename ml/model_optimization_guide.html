Knowledge Distillation & Compilation

Compress large models into smaller ones by transferring knowledge from a teacher to a student model!

📚 What is Knowledge Distillation?

Knowledge Distillation (KD) is a technique where a large, well-trained Teacher model teaches a smaller Student model by transferring knowledge.

  • Teacher Model: Large, powerful model (BERT, GPT, etc.)
  • Student Model: Smaller, faster model (DistilBERT, etc.)
  • Soft Targets: Temperature-scaled probability distributions
  • Knowledge Transfer: Student learns both labels AND teacher's confidence
  • Result: 50-90% size reduction, 5-20x speedup!
💡 Why Soft Targets Matter

Hard target: [0, 1, 0, 0] (one-hot encoded)
Soft target: [0.01, 0.92, 0.05, 0.02] (teacher reveals reasoning)

Student learns not just the answer, but WHY it's correct!

⚙️ How Knowledge Distillation Works

🌡️ Temperature Scaling

Higher T = softer distributions, more knowledge transfer. Typical range: 3-20

📊 KD Loss Function

Combination of CE loss + KL divergence from teacher outputs

🔄 Loss Weighting

α × Task Loss + (1-α) × Distillation Loss (typically α=0.5)

⏱️ Training Time

Similar to fine-tuning, 3-5 epochs usually sufficient

💻 Quick KD Example

from transformers import AutoModelForSequenceClassification, AutoTokenizer
import torch.nn.functional as F

# Load teacher and student
teacher = AutoModelForSequenceClassification.from_pretrained('bert-base-uncased', num_labels=2).eval()
student = AutoModelForSequenceClassification.from_pretrained('distilbert-base-uncased', num_labels=2).train()

# Forward pass
with torch.no_grad():
    teacher_logits = teacher(**batch).logits

student_logits = student(**batch).logits

# Calculate KD loss
T = 3.0  # temperature
kd_loss = F.kl_div(
    F.log_softmax(student_logits/T, dim=-1),
    F.softmax(teacher_logits/T, dim=-1),
    reduction='batchmean'
) * (T * T)

ce_loss = F.cross_entropy(student_logits, labels)
total_loss = 0.5 * ce_loss + 0.5 * kd_loss
total_loss.backward()

🌡️ Temperature Effects Deep Dive

How Temperature Controls Knowledge Transfer:

Temperature Distribution Knowledge Transfer Use Case
T = 1.0 Sharp (peaked) Minimal (standard softmax) No KD, standard training
T = 3-5 Soft (smooth) Moderate (balanced) Most KD applications
T = 10-20 Very soft (flat) Maximum (revealing) Hard tasks, larger models
📊 Temperature Formula

Soft target = softmax(logits / T)

Where:
• Low T: Sharpens distribution, focuses on top predictions
• High T: Smooths distribution, reveals ranking of all classes
• Typical: T ∈ [3, 20] balances information transfer

Fine-tune large models by training only a small fraction of parameters using adapters and LoRA!

⭐ LoRA: Low-Rank Adaptation

Key Idea: Instead of fine-tuning all weights W, fine-tune low-rank decomposition: W + AB^T

🎯
0.1-1%
Trainable Params
💾
10-15x
Memory Reduction
⚡
3-5x
Speed Boost
  • Rank r: Usually 8-64, controls expressiveness (typical: r=8)
  • Alpha: Scaling factor (typical: 16-32)
  • Target modules: Apply to attention queries/values/dense layers
  • Dropout: Regularization to prevent overfitting (typical: 0.05)
  • Result: 0.1-1% of original parameters trainable!
LoRA Formula:

Y = W₀X + (BA)X

Where:
• W₀ = Original frozen weights
• B = (d_out × r) matrix
• A = (r × d_in) matrix
• r = rank (small number like 8)
• Only A and B are trained! 

from peft import LoraConfig, get_peft_model
from transformers import AutoModelForCausalLM

# Load base model
model = AutoModelForCausalLM.from_pretrained('gpt2')

# Configure LoRA
lora_config = LoraConfig(
    r=8,                           # Rank
    lora_alpha=32,                 # Scaling
    target_modules=['c_attn'],     # Apply to attention
    lora_dropout=0.05,
    bias='none',
    task_type='CAUSAL_LM'
)

# Wrap model
peft_model = get_peft_model(model, lora_config)
peft_model.print_trainable_parameters()
# Output: trainable params: 147,456 || all params: 124,439,808 || trainable%: 0.12%

from datasets import Dataset
from transformers import AutoModelForCausalLM, AutoTokenizer, TrainingArguments, Trainer, DataCollatorForLanguageModeling
from peft import LoraConfig, get_peft_model, PeftModel
import torch

# Setup model
model = AutoModelForCausalLM.from_pretrained('gpt2')
tokenizer = AutoTokenizer.from_pretrained('gpt2')
if tokenizer.pad_token is None:
    tokenizer.add_special_tokens({'pad_token': '[PAD]'})
    model.resize_token_embeddings(len(tokenizer))

# Prepare dataset
examples = [
    {"text": "Machine learning is a subset of AI."},
    {"text": "LoRA enables efficient fine-tuning of large models."},
    {"text": "Parameter-efficient methods save memory and time."}
]
dataset = Dataset.from_list(examples)

def tokenize_fn(ex):
    return tokenizer(ex['text'], truncation=True, 
                     padding='max_length', max_length=128)

tokenized = dataset.map(tokenize_fn, batched=True)
tokenized.set_format(type='torch', columns=['input_ids', 'attention_mask'])

# Configure LoRA
lora_config = LoraConfig(
    r=8,
    lora_alpha=32,
    target_modules=['c_attn'],
    lora_dropout=0.05,
    bias='none',
    task_type='CAUSAL_LM'
)

# Wrap model with LoRA
peft_model = get_peft_model(model, lora_config)
peft_model.print_trainable_parameters()

# Training args
training_args = TrainingArguments(
    output_dir='./lora_output',
    num_train_epochs=1,
    per_device_train_batch_size=2,
    gradient_accumulation_steps=4,
    learning_rate=2e-4,
    save_total_limit=1,
    fp16=torch.cuda.is_available()
)

# Trainer
trainer = Trainer(
    model=peft_model,
    args=training_args,
    train_dataset=tokenized,
    tokenizer=tokenizer,
    data_collator=DataCollatorForLanguageModeling(tokenizer, mlm=False)
)

trainer.train()

# Save only LoRA adapters (small!)
peft_model.save_pretrained('./lora_adapter')

# Load into new base model
base_model = AutoModelForCausalLM.from_pretrained('gpt2')
loaded_model = PeftModel.from_pretrained(base_model, './lora_adapter')

# Or merge adapters into base model for deployment
merged_model = peft_model.merge_and_unload()
merged_model.save_pretrained('./lora_merged')

🛡️ OOM-Safe Training Helpers

Handle Out-Of-Memory errors gracefully:

import torch
from torch.cuda.amp import autocast, GradScaler

def oom_safe_backward(loss, optimizer=None, scaler=None, accum_steps=1):
    """Perform backward with OOM detection"""
    try:
        if scaler is not None:
            scaler.scale(loss).backward()
        else:
            loss.backward()
        return True
    except RuntimeError as e:
        if 'out of memory' in str(e).lower():
            print("OOM detected - clearing cache")
            torch.cuda.empty_cache()
            return False
        raise

def smoke_test_tiny():
    """Quick validation of training setup"""
    device = torch.device("cuda" if torch.cuda.is_available() else "cpu")
    model = torch.nn.Sequential(
        torch.nn.Linear(128, 64),
        torch.nn.ReLU(),
        torch.nn.Linear(64, 16)
    ).to(device)
    
    optimizer = torch.optim.Adam(model.parameters(), lr=1e-3)
    scaler = torch.cuda.amp.GradScaler() if torch.cuda.is_available() else None
    
    x = torch.randn(2, 128, device=device)
    y = torch.randn(2, 16, device=device)
    
    with autocast():
        out = model(x)
        loss = torch.nn.functional.mse_loss(out, y)
    
    if not oom_safe_backward(loss, optimizer, scaler):
        return False
    
    if scaler:
        scaler.step(optimizer)
        scaler.update()
    else:
        optimizer.step()
    
    optimizer.zero_grad()
    return True

# Test before full training
if smoke_test_tiny():
    print("✓ Training setup verified!")
⚠️ OOM Prevention Tips
• Use gradient accumulation to simulate larger batches
• Enable gradient checkpointing: reduces activations in memory
• Use mixed precision (FP16): halves memory usage
• Start with small batch size, increase gradually
• Monitor GPU memory with torch.cuda.memory_allocated()

Quantization & Pruning

Reduce model size and increase speed by lowering numerical precision or removing unimportant weights.

🎯 Quantization Methods

Dynamic Quantization

Post-training, CPU-friendly, no calibration needed. Good for NLP models.

Static Quantization

Requires calibration dataset, better accuracy than dynamic, INT8 on GPU.

QAT (Quantization-Aware Training)

Simulate quantization during training. Best accuracy for aggressive quantization.

GPTQ / AWQ

Specialized for LLMs, weight-only quantization preserving accuracy.

✂️ Pruning: Removing Weights

  • Unstructured Pruning: Remove individual weights by magnitude (flexible but irregular)
  • Structured Pruning: Remove entire channels/heads (hardware-friendly speedup)
  • One-shot Pruning: Prune once, then fine-tune
  • Iterative Pruning: Small prune steps with retraining (better accuracy)
  • Sparsity Target: 30-50% typical, aggressive at 80%+ (with accuracy loss)
💡 Pruning Strategy

1. Train baseline model
2. Apply structured pruning (30% sparsity)
3. Fine-tune for 5-10 epochs
4. Quantize to INT8
5. Export and deploy

Result: 6-8x smaller, 10-20x faster!

🔀 Combined Pruning + Quantization Pipeline

Best strategy for maximum compression and speedup:

import torch
import torch.nn as nn
import torch.nn.utils.prune as prune
from transformers import AutoModelForCausalLM

model = AutoModelForCausalLM.from_pretrained('distilgpt2')
model.eval()

# Step 1: Measure baseline
def measure_size(model):
    return sum(p.numel() * p.element_size() for p in model.parameters()) / 1e6

baseline_size = measure_size(model)
print(f'Baseline FP32: {baseline_size:.2f} MB')

# Step 2: Apply 20% structured pruning
for m in model.modules():
    if isinstance(m, nn.Linear):
        prune.ln_structured(m, 'weight', amount=0.2, n=2, dim=0)

# Make pruning permanent
for m in model.modules():
    if hasattr(m, 'weight_mask'):
        prune.remove(m, 'weight')

pruned_size = measure_size(model)
print(f'After Pruning (20%): {pruned_size:.2f} MB')
print(f'  Reduction: {(1 - pruned_size/baseline_size)*100:.1f}%')

# Step 3: Apply INT8 quantization
model_quantized = torch.quantization.quantize_dynamic(
    model,
    {nn.Linear},
    dtype=torch.qint8
)

final_size = measure_size(model_quantized)
print(f'After Pruning + INT8: {final_size:.2f} MB')
print(f'  Total reduction: {(1 - final_size/baseline_size)*100:.1f}%')
print(f'  Compression ratio: {baseline_size/final_size:.2f}x')

# Step 4: Validate inference still works
import torch
test_input = torch.randint(0, 50257, (1, 128))
with torch.no_grad():
    output = model_quantized(test_input)
print('✓ Quantized model works correctly!')

🎯 Structured Pruning Patterns

  • Channel pruning: Remove entire channels from convolutional layers
  • Neuron pruning: Remove neurons from fully connected layers
  • Head pruning: Remove attention heads (specific to transformers)
  • Layer pruning: Remove entire transformer blocks
  • Block-structured: Prune blocks of weights for hardware efficiency
🎯 Iterative Pruning Best Practice

1. Train baseline → evaluate
2. Prune 10% → fine-tune 2 epochs
3. Prune 10% more → fine-tune 2 epochs
4. Repeat until accuracy loss acceptable

Result: Better accuracy retention than one-shot pruning at same sparsity!

💾 Paged Attention: The Core Innovation

Problem: KV cache memory usage quadratic with sequence length, causing memory waste.

Solution: Treat KV cache like virtual memory - store in pages, evict unused ones!

  • Memory fragmentation fix: Pre-allocate KV cache in blocks
  • Waste reduction: 15-30% less memory for sequences with padding
  • Higher throughput: Fit more requests in batch
  • Seamless batching: Different sequence lengths work easily
  • No accuracy loss: Completely transparent optimization
🧠 Paged Attention Analogy

Without Paged Attention: Allocate contiguous memory block for max length (wastes space for short sequences)

With Paged Attention: Allocate pages (like RAM), use only what you need, free unused pages for other requests

Result: Higher GPU memory utilization = more concurrent requests = better throughput!

📈 Benchmarking & Profiling

  • Memory profiling: torch.cuda.memory_allocated(), torch.cuda.max_memory_allocated()
  • Speed benchmarking: Warm-up runs, then measure 100 iterations
  • Accuracy metrics: Always validate on test set after each optimization
  • End-to-end testing: Profile entire pipeline, not just individual components
  • Hardware-specific tuning: Batch size, gradient accumulation depend on GPU type 

import torch
import time

def benchmark_model(model, data, name, num_runs=100):
    # Warm up
    for _ in range(10):
        _ = model(data)
    
    # Benchmark
    torch.cuda.reset_peak_memory_stats()
    start = time.time()
    
    for _ in range(num_runs):
        _ = model(data)
    
    end = time.time()
    avg_time = (end - start) / num_runs
    peak_memory = torch.cuda.max_memory_allocated() / 1e9
    
    print(f"{name}")
    print(f"  Time/iter: {avg_time*1000:.2f}ms")
    print(f"  Throughput: {1/avg_time:.0f} iter/sec")
    print(f"  Peak Memory: {peak_memory:.2f}GB")
    print()

⚠️ Common Optimization Pitfalls

❌ Pitfall 1: Over-aggressive Quantization
INT4 without proper calibration → severe accuracy loss. Always test on validation set!
❌ Pitfall 2: Ignoring Warmup
First few iterations slower due to compilation/allocation. Warm up before benchmarking!
❌ Pitfall 3: Single-metric Optimization
Optimizing only for speed sacrifices accuracy. Measure end-to-end performance!
❌ Pitfall 4: Not Testing Edge Cases
Test on maximum sequence length, batch sizes, and rare inputs before deployment!
