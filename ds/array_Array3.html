💭 Problem: Sort array of 0s, 1s, 2s in-place without library sort.

🚩 MEMORY TRICK: "L-M-H" (Low-Mid-High) → Three zones like a flag!
💡 KEY INSIGHT: Don't sort! Just partition into 3 zones using pointers.
Pattern Recognition:
[0s ZONE] | [1s ZONE] | [2s ZONE] ← 3 separate areas!
Algorithm: 3-Way Partition
  • 🔴 low: Right boundary of 0s (move right after swapping 0)
  • 🟡 mid: Scanner pointer (checks each element)
  • 🔵 high: Left boundary of 2s (move left after swapping 2)
Pro Tip: When nums[mid]=0, move BOTH low & mid. When nums[mid]=2, move only high. This avoids re-checking!
Gotcha: Don't increment mid when swapping with high! You need to check that element again.
⏱️ Time
O(n)
💾 Space
O(1)
Code Solution
def sort_colors(nums):
    low, mid, high = 0, 0, len(nums) - 1
    
    while mid <= high:
        if nums[mid] == 0:
            nums[low], nums[mid] = nums[mid], nums[low]
            low += 1; mid += 1  # Both move!
        elif nums[mid] == 1:
            mid += 1  # Just move mid
        else:
            nums[mid], nums[high] = nums[high], nums[mid]
            high -= 1  # Only high moves!
🎬 Step-by-Step Trace: [2,0,2,1,1,0]
TRACE: Start: [2,0,2,1,1,0], low=0, mid=0, high=5 Step1: nums[0]=2 (swap with high) → [0,0,2,1,1,2], high=4 Step2: nums[0]=0 (swap with low) → [0,0,2,1,1,2], low=1, mid=1 Step3: nums[1]=0 (already at low) → [0,0,2,1,1,2], low=2, mid=2 ... Final: [0,0,1,1,2,2] ✅
❓ Quick Check: What happens when nums[mid]=2?
🎯 Only decrement high (don't increment mid)
❌ Increment both mid and high
❌ Only increment mid
🚩 L-M-H: Like a Traffic Light! 🚦
🔴 Red (0) on LEFT | 🟡 Yellow (1) MIDDLE | 🟢 Green (2) RIGHT

💭 Problem: Count subarrays where product < k using sliding window.

🪟 MEMORY: "Expand Right, Contract Left" → Classic Sliding Window Dance!
💡 KEY: For each right position, ALL subarrays in window are valid!
Math Pattern:
At position right: Valid subarrays = right - left + 1
Genius Move: Once window product ≥ k, shrink from LEFT to reduce product. Don't reset, just slide!
⏱️ Time
O(n)
💾 Space
O(1)
Code Solution
def num_subarray_product(nums, k):
    if k <= 1: return 0
    
    prod, left, res = 1, 0, 0
    
    for right in range(len(nums)):
        prod *= nums[right]  # Expand
        
        while prod >= k and left <= right:
            prod //= nums[left]  # Contract
            left += 1
        
        res += right - left + 1  # Count all valid!
    
    return res
🎪 SLIDING WINDOW SONG: 🎵
"Expand, expand... STOP! Contract, contract... GO!"

💭 Problem: Find duplicate in O(1) space without modifying array.

🐢🐰 MEMORY: "Tortoise & Hare" → Classic cycle detection algorithm!
💡 GENIUS: Treat array as linked list! Values = indices → natural cycle!
Cycle Insight:
Duplicate number = cycle entry point! 🎯
Two Phase: Phase 1: Find collision point. Phase 2: Find cycle entry!
Trap: Don't use HashSet (violates O(1) space requirement)!
⏱️ Time
O(n)
💾 Space
O(1) 🌟
Code Solution
def find_duplicate(nums):
    # Phase 1: Find meeting point
    tortoise = hare = nums[0]
    while True:
        tortoise = nums[tortoise]
        hare = nums[nums[hare]]
        if tortoise == hare: break
    
    # Phase 2: Find cycle entry
    ptr1 = nums[0]
    ptr2 = tortoise
    while ptr1 != ptr2:
        ptr1 = nums[ptr1]
        ptr2 = nums[ptr2]
    
    return ptr1
🏃🐢 RACE TRACK ANALOGY: 🏁
Hare = 2x speed, Tortoise = 1x speed → They ALWAYS meet in a cycle!

💭 Problem: Find max product subarray (handle negatives!).

⚡ MEMORY: "Negative × Negative = POSITIVE!" → Track both max & min!
💡 KEY: A "bad" minimum can become the best maximum after hitting negative!
State Machine:
Two states: cur_max & cur_min. Swap when negative found!
Swap Magic: When num < 0, swap cur_max ↔ cur_min. The minimum suddenly becomes maximum!
⏱️ Time
O(n)
💾 Space
O(1)
Code Solution
def max_product(nums):
    res = cur_max = cur_min = nums[0]
    
    for n in nums[1:]:
        if n < 0:  # Negative coming!
            cur_max, cur_min = cur_min, cur_max  # SWAP!
        
        cur_max = max(n, cur_max * n)
        cur_min = min(n, cur_min * n)
        res = max(res, cur_max)
    
    return res
🔄 STATE FLIP DANCE: ↔️
See negative? → SWAP max & min → Minimum becomes tomorrow's maximum!

💭 Problem: Find minimum swaps needed to sort array using cycle detection.

🔁 MEMORY: "Follow the cycles!" → Each cycle needs (size-1) swaps
💡 KEY: Graph each element to its correct position, trace cycles!
Cycle Formula: For k-size cycle → k-1 swaps needed!
⏱️ Time
O(n log n)
💾 Space
O(n)
Code Solution
def min_swaps(arr):
    # Create (value, index) pairs and sort by value
    n = len(arr)
    arrPos = [(arr[i], i) for i in range(n)]
    arrPos.sort()
    
    visited = [False] * n
    swaps = 0
    
    for i in range(n):
        if visited[i] or arrPos[i][1] == i:
            continue
        
        # Count cycle length
        cycle_len = 0
        j = i
        while not visited[j]:
            visited[j] = True
            j = arrPos[j][1]
            cycle_len += 1
        
        if cycle_len > 1:
            swaps += (cycle_len - 1)
    
    return swaps

💭 Problem: Count pairs (i,j) where i < j but arr[i]> arr[j].

🔀 MEMORY: "Count while merging!" → Merge sort counts inversions automatically
💡 KEY: When left[i] > right[j], ALL remaining left are inversions!
Trick: Add (len(left)-i) for each inversion found!
⏱️ Time
O(n log n)
💾 Space
O(n)
Code Solution
def count_inversions(arr):
    # Merge sort approach
    def merge_count(left, right):
        i = j = 0
        count = 0
        merged = []
        
        while i < len(left) and j < len(right):
            if left[i] <= right[j]:
                merged.append(left[i])
                i += 1
            else:
                merged.append(right[j])
                count += len(left) - i  # Key: ALL remaining left > right[j]
                j += 1
        
        merged.extend(left[i:])
        merged.extend(right[j:])
        return merged, count
    
    def sort_count(arr):
        if len(arr) <= 1:
            return arr, 0
        
        mid = len(arr) // 2
        left, lc = sort_count(arr[:mid])
        right, rc = sort_count(arr[mid:])
        merged, mc = merge_count(left, right)
        
        return merged, lc + rc + mc
    
    _, inversions = sort_count(arr)
    return inversions

💭 Problem: Find missing number 0-n using XOR trick.

⊕ MEMORY: "a ⊕ a = 0, a ⊕ 0 = a" → Duplicates cancel!
💡 GENIUS: XOR all indices with all elements → Duplicate cancels → Missing remains!
No Overflow: XOR works for large numbers (unlike sum approach)!
⏱️ Time
O(n)
💾 Space
O(1)
Code Solution
def missing_number(nums):
    # XOR magic: a ^ a = 0, a ^ 0 = a
    res = len(nums)
    
    for i, num in enumerate(nums):
        res ^= i ^ num
    
    return res
🎪 XOR SONG: 🎵
"Self XOR zero, other XOR remains... Missing shows up in the end!"

💭 Problem: Check if n flowers can be planted (no adjacency).

🌼 MEMORY: "Lonely flowers only!" → Need empty neighbors!
💡 KEY: 3 conditions: current empty AND left empty AND right empty!
Greedy Works: Plant whenever possible - optimal strategy!
⏱️ Time
O(n)
💾 Space
O(1)
Code Solution
def can_place_flowers(flowerbed, n):
    count = 0
    i = 0
    
    while i < len(flowerbed):
        if (flowerbed[i] == 0 and
            (i == 0 or flowerbed[i-1] == 0) and
            (i == len(flowerbed)-1 or flowerbed[i+1] == 0)):
            
            flowerbed[i] = 1
            count += 1
        
        i += 1
    
    return count >= n

💭 Problem: Arrange numbers to form largest possible number.

🎯 MEMORY: "Compare concatenations!" → "xy" vs "yx"?
💡 GENIUS: Custom comparator: if (x+y > y+x) → x comes first!
Trap: Watch "00...0" case → return "0"!
⏱️ Time
O(n log n)
💾 Space
O(n)
Code Solution
from functools import cmp_to_key

def largest_number(nums):
    nums = list(map(str, nums))
    
    def cmp(x, y):
        if x + y > y + x:
            return -1
        elif x + y < y + x:
            return 1
        else:
            return 0
    
    nums.sort(key=cmp_to_key(cmp))
    res = ''.join(nums)
    
    return '0' if res[0] == '0' else res
💬 DECISION RULE: 📝
"102" vs "210" → "210" bigger → put "2" first!

💭 Problem: Find max length subarray with equal 0s and 1s.

🔑 MEMORY: "Convert 0→-1, find sum=0!" → Prefix sum magic!
💡 KEY: If (current_sum - k) exists → found subarray!
Trick: Equal count = sum zero!
⏱️ Time
O(n)
💾 Space
O(n)
Code Solution
def find_max_length(nums):
    count = 0
    max_len = 0
    seen = {0: -1}  # sum -> first index
    
    for i, num in enumerate(nums):
        # Convert 0 to -1, so equal count gives sum=0
        count += 1 if num == 1 else -1
        
        if count in seen:
            # Found subarray with sum=0
            max_len = max(max_len, i - seen[count])
        else:
            seen[count] = i
    
    return max_len
🧮 EQUATION TRICK: ➕
If count seen before, subarray between has equal 0s & 1s!

💭 Problem: Find max average of k consecutive elements.

🪟 MEMORY: "Sliding window of fixed size!" → Add new, drop old!
💡 KEY: No need to recalculate sum → Just add new, subtract old!
O(1) Trick: window_sum += nums[i] - nums[i-k]!
⏱️ Time
O(n)
💾 Space
O(1)
Code Solution
def find_max_average(nums, k):
    window_sum = sum(nums[:k])
    max_sum = window_sum
    
    for i in range(k, len(nums)):
        window_sum += nums[i] - nums[i-k]
        max_sum = max(max_sum, window_sum)
    
    return max_sum / k

💭 Problem: Find longest subarray forming mountain (up then down).

⛰️ MEMORY: "Peak detection!" → Find peak, expand left & right!
💡 KEY: Peak = arr[i-1] < arr[i]> arr[i+1]!
Efficient: Each element visited once → O(n)!
⏱️ Time
O(n)
💾 Space
O(1)
Code Solution
def longest_mountain(arr):
    if len(arr) < 3:
        return 0
    
    ans = 0
    i = 1
    
    while i < len(arr) - 1:
        if arr[i-1] < arr[i] > arr[i+1]:
            l = i - 1
            while l > 0 and arr[l-1] < arr[l]:
                l -= 1
            
            r = i + 1
            while r < len(arr) - 1 and arr[r] > arr[r+1]:
                r += 1
            
            ans = max(ans, r - l + 1)
            i = r + 1
        else:
            i += 1
    
    return ans
/\ MOUNTAIN SHAPE: /\
UP (strictly increasing) + DOWN (strictly decreasing) = Valid mountain!

💭 Problem: Check if array is monotonic (all inc OR all dec).

🎯 MEMORY: "One direction only!" → Inc OR Dec, never both!
💡 KEY: Two flags: inc & dec. Set false when condition breaks!
Result: Return (inc OR dec) at end!
⏱️ Time
O(n)
💾 Space
O(1)
Code Solution
def is_monotonic(nums):
    inc = dec = True
    
    for i in range(1, len(nums)):
        if nums[i] > nums[i-1]:
            dec = False
        if nums[i] < nums[i-1]:
            inc = False
    
    return inc or dec

💭 Problem: Find index where left sum = right sum.

⚖️ MEMORY: "Balance point!" → Weights equal on both sides!
💡 GENIUS: right_sum = total - left_sum - current!
One Pass: Accumulate left_sum, calculate right on the fly!
⏱️ Time
O(n)
💾 Space
O(1)
Code Solution
def pivot_index(nums):
    total = sum(nums)
    left_sum = 0
    
    for i, x in enumerate(nums):
        if left_sum == total - left_sum - x:
            return i
        left_sum += x
    
    return -1

def replace_elements(arr):

💭 Problem: Find duplicates in O(1) space using array indices.

🚩 MEMORY: "Mark with negativity!" → visited = negative!
💡 GENIUS: Use array indices 0-n as hash → Mark with negative!
Two Passes: First mark, second collect duplicates!
⏱️ Time
O(n)
💾 Space
O(1)
Code Solution
def find_duplicates(nums):
    res = []
    
    for n in nums:
        idx = abs(n) - 1
        
        if nums[idx] < 0:
            res.append(abs(n))
        else:
            nums[idx] = -nums[idx]
    
    return res
🔢 MARKING SYSTEM: 🚩
Positive = unvisited, Negative = visited = duplicate!

💭 Problem: Find next greater element in circular array.

📚 MEMORY: "Monotonic stack!" → Process 2×n elements!
💡 KEY: Iterate 2×n times to handle circular. Stack stores indices!
Stack Trick: Pop while current > stack top!
⏱️ Time
O(n)
💾 Space
O(n)
Code Solution
def next_greater_elements(nums):
    n = len(nums)
    res = [-1] * n
    stack = []
    
    for i in range(2 * n):
        while stack and nums[stack[-1]] < nums[i % n]:
            res[stack.pop()] = nums[i % n]
        
        if i < n:
            stack.append(i)
    
    return res
🎪 CIRCULAR TRICK: 🔁
Process 2×n elements → Circular becomes 2 linear scans!

💭 Problem: Count subarrays with specific sum k.

🔑 MEMORY: "Check if (prefix - k) exists!" → Prefix sum magic!
💡 GENIUS: If (current_sum - k) seen before → found one!
Count Storage: Store frequency of each sum!
⏱️ Time
O(n)
💾 Space
O(n)
Code Solution
def subarray_sum(nums, k):
    count = 0
    prefix_sum = 0
    sums = {0: 1}  # sum -> frequency
    
    for num in nums:
        prefix_sum += num
        
        if prefix_sum - k in sums:
            count += sums[prefix_sum - k]
        
        sums[prefix_sum] = sums.get(prefix_sum, 0) + 1
    
    return count
🧮 EQUATION TRICK: ➕
If prefix_sum - k = old_sum, then subarray between has sum k!

💭 Problem: Find all elements appearing more than n/3 times.

🎯 MEMORY: "Max 2 candidates!" → Pigeonhole principle!
💡 GENIUS: At most 2 elements > n/3. Track candidates & counts!
Two Phases: Phase 1: Find candidates. Phase 2: Verify!
Gotcha: Must verify in second pass!
⏱️ Time
O(n)
💾 Space
O(1)
Code Solution
def majority_element_n3(nums):
    if not nums:
        return []
    
    cand1 = cand2 = None
    cnt1 = cnt2 = 0
    
    # Find candidates
    for n in nums:
        if cand1 == n:
            cnt1 += 1
        elif cand2 == n:
            cnt2 += 1
        elif cnt1 == 0:
            cand1, cnt1 = n, 1
        elif cnt2 == 0:
            cand2, cnt2 = n, 1
        else:
            cnt1 -= 1
            cnt2 -= 1
    
    # Verify candidates
    res = []
    for c in (cand1, cand2):
        if c is not None and nums.count(c) > len(nums) // 3:
            res.append(c)
    
    return res
🎪 PIGEONHOLE LAW: 🕳️
n holes, n+2 pigeons → At most 2 pigeons appear > 3 times!
