Problem: Find the contiguous subarray with the largest sum.

💡 Key Insight: At each position, decide: keep adding to current sum or start fresh from current element?
Approach: Greedy Single Pass
  • Track current sum at each position
  • At each element, take max(element, current_sum + element)
  • Update global maximum whenever current exceeds it
  • Complete in single pass!
⏱️ Time
O(n)
💾 Space
O(1)
Solution
def max_subarray(nums):
    if not nums:
        return 0
    max_sum = curr = nums[0]
    for num in nums[1:]:
        curr = max(num, curr + num)
        max_sum = max(max_sum, curr)
    return max_sum

Problem: Return indices of two numbers that add up to target value.

💡 Key Insight: Use HashMap! For each number, check if its complement exists.
Approach: Hash Map
  • Create hash map to store {number → index}
  • For each number, calculate complement = target - number
  • Check if complement exists in map
  • Return indices immediately on match
⏱️ Time
O(n)
💾 Space
O(n)
Solution
def two_sum(nums, target):
    seen = {}
    for i, num in enumerate(nums):
        complement = target - num
        if complement in seen:
            return [seen[complement], i]
        seen[num] = i
    return []
📋 Example: [2,7,11,15], target=9
Step 1: num=2, complement=7, seen={2:0} Step 2: num=7, complement=2, FOUND! Answer: [0, 1] ✓

Strings 2

Problem: Return length of longest substring with no repeating characters.

💡 Key Insight: Use sliding window! Expand right, contract left when duplicate found.
Approach: Sliding Window
  • Maintain window with left and right pointers
  • Track seen characters in Set
  • Expand: add characters with right pointer
  • Contract: shrink from left when duplicate found
⏱️ Time
O(n)
💾 Space
O(m)
Solution
def longest_substring(s):
    seen = set()
    left = 0
    max_len = 0
    for right, ch in enumerate(s):
        while ch in seen:
            seen.remove(s[left])
            left += 1
        seen.add(ch)
        max_len = max(max_len, right - left + 1)
    return max_len

Problem: Group strings into lists of anagrams.

💡 Key Insight: Anagrams have identical sorted characters as key!
Approach: Sorting
  • Sort characters in each string for canonical form
  • Use sorted string as dictionary key
  • Anagrams group automatically under same key
⏱️ Time
O(n×k logk)
💾 Space
O(n×k)
Solution
from collections import defaultdict

def group_anagrams(strs):
    groups = defaultdict(list)
    for s in strs:
        key = tuple(sorted(s))
        groups[key].append(s)
    return list(groups.values())

LinkedList 2

Problem: Merge two sorted linked lists into one sorted list.

💡 Key Insight: Two pointers, one for each list. Compare and attach the smaller.
Approach: Two Pointer Merge
  • Create dummy node to simplify edge cases
  • Use two pointers at heads of both lists
  • Compare and attach smaller node
  • Append remaining list when one exhausted
⏱️ Time
O(n+m)
💾 Space
O(1)
Solution
def merge_two_lists(l1, l2):
    dummy = ListNode(0)
    curr = dummy
    while l1 and l2:
        if l1.val < l2.val:
            curr.next = l1
            l1 = l1.next
        else:
            curr.next = l2
            l2 = l2.next
        curr = curr.next
    curr.next = l1 or l2
    return dummy.next

Problem: Remove the nth node from the end of the list.

💡 Key Insight: Use two pointers n steps apart! Find node before target.
Approach: Two Pointer Gap
  • Create dummy node pointing to head
  • Fast pointer moves n steps ahead
  • Both move until fast reaches end
  • Slow pointer is just before target
⏱️ Time
O(n)
💾 Space
O(1)
Solution
def remove_nth(head, n):
    dummy = ListNode(0, head)
    fast = slow = dummy
    for _ in range(n):
        fast = fast.next
    while fast.next:
        fast = fast.next
        slow = slow.next
    slow.next = slow.next.next
    return dummy.next

Stacks & Queues 2

Problem: Find next greater element for each element in circular array.

💡 Key Insight: Use monotonic stack! Store indices, iterate array twice for circular effect.
Approach: Monotonic Stack
  • Use stack to track unmatched elements (indices)
  • Iterate array twice (simulate circular)
  • Pop elements smaller than current
  • Current becomes their next greater
⏱️ Time
O(n)
💾 Space
O(n)
Solution
def next_greater(nums):
    n = len(nums)
    res = [-1] * n
    stack = []
    for i in range(2 * n):
        num = nums[i % n]
        while stack and nums[stack[-1]] < num:
            res[stack.pop()] = num
        if i < n:
            stack.append(i)
    return res

Problem: Return maximum for each sliding window of size k.

💡 Key Insight: Use deque! Maintain decreasing order. Front is always max!
Approach: Deque
  • Maintain deque in decreasing order of values
  • Remove indices outside window
  • Remove smaller elements from back
  • Front of deque is always max
⏱️ Time
O(n)
💾 Space
O(k)
Solution
from collections import deque

def max_sliding_window(nums, k):
    q = deque()
    res = []
    for i, num in enumerate(nums):
        while q and nums[q[-1]] < num:
            q.pop()
        q.append(i)
        if q[0] == i - k:
            q.popleft()
        if i >= k - 1:
            res.append(nums[q[0]])
    return res

Trees 2

Problem: Perform inorder traversal using iteration (stack).

💡 Key Insight: Use stack to simulate recursion. Go left until null, then process.
Approach: Stack Simulation
  • Use stack to store nodes while going left
  • Go left until null, push all nodes
  • Pop, add to result, go right
  • Repeat until stack empty and current null
⏱️ Time
O(n)
💾 Space
O(h)
Solution
def inorder_traversal(root):
    stack, res = [], []
    curr = root
    while curr or stack:
        while curr:
            stack.append(curr)
            curr = curr.left
        curr = stack.pop()
        res.append(curr.val)
        curr = curr.right
    return res

Problem: Find diameter (longest path between any two nodes).

💡 Key Insight: At each node, diameter = left_height + right_height. Track maximum!
Approach: DFS with Height
  • DFS to calculate depth of each subtree
  • At each node, diameter = left_depth + right_depth
  • Track maximum diameter across all nodes
  • Return height for parent calculations
⏱️ Time
O(n)
💾 Space
O(h)
Solution
def diameter(root):
    diameter = 0
    def depth(node):
        nonlocal diameter
        if not node:
            return 0
        left = depth(node.left)
        right = depth(node.right)
        diameter = max(diameter, left + right)
        return 1 + max(left, right)
    depth(root)
    return diameter
