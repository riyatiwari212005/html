🛍️ Meta GrokNet: Product Understanding (2020)

⚙️ Extreme Multi-Task Learning: 83 Tasks
🎯 Challenge: Predict 83 different attributes simultaneously from a single product image!

Meta's GrokNet revolutionizes product understanding by training a single CNN to predict:

├── 16 Category Heads (Hierarchical) │ ├── Main category (fashion, electronics, home, etc.) │ ├── Subcategory 1-5 │ └── Sub-subcategory │ ├── 45 Attribute Heads (Multi-label) │ ├── Color taxonomy (primary, secondary, accents) - 12 heads │ ├── Material (leather, cotton, polyester, etc.) - 10 heads │ ├── Size & fit (XS, S, M, L, XL, etc.) - 5 heads │ ├── Condition (new, like new, good, fair, poor) - 5 heads │ └── Style/occasion (casual, formal, athletic, etc.) - 8 heads │ ├── 1 Embedding Head │ └── 256-dimensional product embedding for similarity search │ ├── 1 Search Head │ └── Predict: "Will this match query [query_ids]?" │ └── 20+ Other Heads ├── Price prediction ├── Seasonality ├── Trend score └── Various other predictions

The Multi-Task Learning Problem

❌ Traditional Approach:
  • Manually tune weights for 83 losses
  • When one task improves, others degrade
  • Zero-sum optimization game
  • Some tasks: Easy (converge quickly)
  • Other tasks: Hard (slow to improve)

✨ Task Uncertainty Weighting: The Solution

🔴 Easy Task ← Small σ → High Weight → ⚠️ 🟢 Hard Task ← Large σ → Low Weight → ✅ 🔵 Automatic Balancing ← Learn σ → All Tasks Improve
Kendall 2017 - Task Uncertainty Loss:

L_total = Σᵢ (1/σᵢ²) × L_i + log(σᵢ²)

Where:
• σᵢ² = learned uncertainty parameter for task i
• 1/σᵢ² = automatic weight (inverse of variance)
• log(σᵢ²) = regularization term preventing σ → ∞
💡 How It Works:
  • Small σᵢ (certain): 1/σᵢ² is large → Task receives high loss weight → Network focuses on it
  • Large σᵢ (uncertain): 1/σᵢ² is small → Task receives low weight → Network relaxes, lower priority
  • Example: σ=0.01 → weight=10,000x! | σ=10 → weight=0.01x
  • Result: All 83 tasks improve simultaneously without manual tuning!

🧬 Embedding Learning & Compression

📸 Product Image → 🧠 CNN → 📊 256-D Vector → ⚡ Normalized → 🎯 Similarity Search

Why Embeddings Matter

Traditional Keyword Matching:
  • Problem: "shoe" ≠ "sneaker" ≠ "athletic footwear"
  • Result: Miss relevant products (synonym mismatch)
Semantic Embeddings:
  • Red running shoes → specific vector in 256-D space
  • Blue running shoes → nearby vector (similar features)
  • Search: Find nearest neighbors!
  • Result: Semantic similarity → geometric proximity

🎲 Triplet Loss: Learning Quality Embeddings

Goal: Similar products close together, dissimilar products far apart

Triplet Loss Formula:
L = max(0, d(anchor, positive) - d(anchor, negative) + margin)

Where:
• Anchor = reference product
• Positive = same product (different angle)
• Negative = different product
• margin = 0.5 (ensures negative at least 0.5 units further)

🗜️ Embedding Compression: Quantization

Memory Challenge: 256-D float32 = 1024 bytes/product × 1B products = 10 TB memory!
Solution: Quantization (Float32 → Int8)
  • 256 × 4 bytes → 256 × 1 byte = 4x compression!
  • Process: Min/max range → [0, 255] → round to int
  • Example: 0.567 × 255 = 144.5 → 145 (recovered: 145/255 = 0.569)
  • Per-element error: 0.002 (tiny!)
  • Total effect: ~3% distance distortion (acceptable!)

Batch Composition Strategy:

🌦️ Google MetNet: Weather Forecasting (2020)

🎯 Challenge: Predict precipitation with probabilities 0-8 hours ahead using radar, satellite, and meteorological data!

🔄 ConvLSTM: Combining CNNs and RNNs

Why Standard LSTM Fails for Weather

Standard LSTM: ├── Input: 1D sequence [temp_1, temp_2, ..., temp_t] ├── Processing: Time-only (1D temporal) ├── Problem: LOSES spatial structure! │ Weather Challenge: ├── Input: 10×10 grid of temperatures over 5 timesteps ├── Needs BOTH spatial AND temporal ├── Naive approach: Flatten 10×10 → 100-D vector │ └── Neighbors (4,4)-(4,5) treated same as (1,1)-(9,9) ✗ │ Solution: ConvLSTM └── LSTM cell with convolutions instead of matrix multiply!
🖼️ Spatial Grid → ConvLSTM → ⏰ Temporal History → 🌦️ Weather Prediction

ConvLSTM Cell Operations

Standard LSTM:
y = f(x)

ConvLSTM (Replacement):
i_t = σ(Conv(X_t, W_xi) + Conv(H_{t-1}, W_hi))
f_t = σ(Conv(X_t, W_xf) + Conv(H_{t-1}, W_hf))
o_t = σ(Conv(X_t, W_xo) + Conv(H_{t-1}, W_ho))
c_t = tanh(Conv(X_t, W_xc) + Conv(H_{t-1}, W_hc))
h_t = o_t ⊙ tanh(c_t)

Key: All matrix multiplies replaced with convolutions!
✨ Why ConvLSTM Works:
  • Preserves spatial structure naturally
  • Local connectivity: Each cell sees neighbors (3×3 kernel)
  • Memory: Long-term temporal dependencies
  • Perfect for: video, weather, optical flow!

🔍 Attention Mechanisms for Long-Range Dependencies

The Problem: Receptive Field Limitations

ConvLSTM with 3×3 kernel:
  • After 1 layer: 3×3 neighborhood
  • After 2 layers: 5×5 neighborhood
  • After N layers: (2N+1)×(2N+1) neighborhood
  • Problem: To reach 512×512 image need 250+ layers! ❌
Solution: Axial Attention (Efficient Spatial Attention)
  • Allows network to directly reference distant locations
  • Complexity: O(H²W²) → O(HW(H+W)) [260x reduction!]
  • Memory: ~1GB (vs 274GB for full attention)

🎯 Axial Attention Process

Axial Attention Split into 2 Passes: Pass 1: HORIZONTAL ATTENTION ├── For each pixel (h, w): Attend to all (h, 0..W) ├── Complexity: O(H × W × W) manageable! ├── Captures: Horizontal patterns (weather fronts) Pass 2: VERTICAL ATTENTION ├── For each pixel (h, w): Attend to all (0..H, w) ├── Same complexity as horizontal ├── Captures: Vertical development (storm depth) RESULT: Complementary information ├── Together capture 2D structure ├── Each pixel connects to ALL others in its row/column ├── Long-range dependencies captured! └── No blind spots

🌧️ Prediction Approach: Probabilistic vs Deterministic

Regression (Single Value):
  • Predict: rainfall amount (mm)
  • Output: Single number (e.g., 2.5 mm)
  • Problem: Misses uncertainty (chaotic weather!)
  • Use case: Climate (long-term trends)
Classification (Categories):
  • Predict: Binary (rain/no rain) or categories (light/moderate/heavy)
  • Advantage: Matches real use ("Will it rain?")
  • Loss: Information (heavy ≠ light)
🌟 Probabilistic (MetNet's Approach):
  • Predict: Probability distribution over rainfall
  • Output: P(0 mm)=40%, P(light)=30%, P(moderate)=20%, P(heavy)=10%
  • Advantages: Captures uncertainty, prescriptive detail, enables risk-sensitive decisions
  • Disadvantages: Requires more data, harder to optimize

🎓 Loss Function & Training

Probabilistic Loss (Cross-Entropy):
L = -Σ y_i × log(ŷ_i)

Where y is one-hot encoded true precipitation class

Multi-timestep Training:
L_total = Σ_(t=1 to 240) weight(t) × L_at_time(t)
weight(t) increases with t (care more about far future)
Training Configuration:
  • Optimizer: Adam (lr=0.0001, gradient clipping=5.0)
  • Data: 2 years MRMS data (2017-2019), ~39M samples
  • Augmentation: Temporal shifts, spatial shifts
  • Batch Size: 32 (GPU memory limited)
  • GPU: NVIDIA Tesla V100 (32GB)
  • Training Time: ~2 weeks on 1 GPU

🧮 Key Concepts & Techniques

📚 Multi-Task Learning

Definition: Train single model to predict multiple outputs simultaneously
  • Traditional: Train separate model for each task
  • Multi-task: Share representation + task-specific heads
  • Benefit: Shared features improve all tasks
  • Challenge: Balancing different task losses

🧬 Embeddings & Vector Spaces

What: Learned dense vector representations
  • Map high-dimensional data to lower dimension
  • Preserve semantic similarity as geometric proximity
  • Enable similarity search (nearest neighbors)

⚡ Loss Functions

Cross-Entropy (Classification):
L = -Σ y_i × log(ŷ_i)

Mean Squared Error (Regression):
L = (1/n) × Σ(y_i - ŷ_i)²

Triplet Loss (Embeddings):
L = max(0, d(anchor, positive) - d(anchor, negative) + margin)
