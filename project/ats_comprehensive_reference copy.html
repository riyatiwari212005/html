🛠️ Microservices Architecture Overview

🎯 Core Purpose: 15 loosely-coupled microservices form a comprehensive Applicant Tracking System (ATS) with AI-powered recruitment capabilities

Architecture Layers

├── Authentication & Authorization (Layer 1) │ ├── Login Service (8500) │ └── RBAC Service (8504) │ ├── Core Data Management (Layer 2) │ ├── Candidate Service (8503) │ ├── Company Service (8506) │ ├── Job Service (8507) │ └── Job Description Service (8084, 8509) │ ├── Document Processing (Layer 3) │ ├── File Handler Service (8501) │ ├── Resume Analyzer (8512) │ └── Web Scraping Service (8502) │ ├── AI Processing (Layer 4) │ ├── Candidate Recommendation (AI-powered matching) │ ├── Skill Assessment Service (8084) │ └── Job Agents Service (8510) │ ├── Orchestration (Layer 5) │ ├── Pipeline Service (8505) │ └── Status Service (8515) │ └── Infrastructure ├── MySQL Database ├── Redis Cache ├── Celery Task Queue └── Docker Containers

Service Communication Flow

📝 Resume Upload → 📂 File Handler → 🧠 Resume Analyzer → 💾 Candidate Service → 🤖 Recommendation Engine
📋 Job Posted → 🔍 Web Scraper → 💼 Job Service → 📊 Pipeline Orchestration → ✉️ Job Agents (Marketing)

🔄 Jenkins CI/CD Pipeline Architecture

🎯 Purpose: Automated build, test, and deployment for all 13 ATS microservices with Docker containerization
1
Active Branch (QA)
7
Pipeline Stages
0
Concurrent Builds

📊 Pipeline Flow Overview

🔗 GitHub Webhook → 🔐 Guard Branch → ⚙️ Init → 📥 Checkout →
🐳 Build Image → 🛑 Stop Container → 🚀 Run Container → ✅ Done

🎯 Pipeline Stages (Detailed)

1️⃣ Guard Branch - Validation

✓ Checks if branch is 'qa'
✓ Skips non-qa commits
✓ Prevents unexpected deployments

2️⃣ Init - Environment Setup

✓ Loads environment variables
✓ Loads Jenkins credentials
✓ Sets APP_ENV, APP_PORT

3️⃣ Checkout - Source Code

✓ Clones from repository
✓ Fetches latest changes

4️⃣ Build Image - Docker Build

✓ Creates Docker image
✓ Tags with environment
✓ Runs on 'qa' branch only

5️⃣ Stop Old Container - Cleanup

✓ Gracefully stops container
✓ Removes old resources
✓ Ignores if not running

6️⃣ Run Container - Deployment

✓ Launches new container
✓ Injects credentials
✓ Mounts volumes
✓ Connects to network
✓ Maps ports
✓ Auto-restart on failure

7️⃣ Post - Reporting & Cleanup

✓ Generates build report
✓ Sends notifications
✓ Performs cleanup

⚙️ Build Configuration

🔒 Concurrent Builds: DISABLED

Prevents simultaneous deployments of the same service, ensuring sequential and ordered execution

🌿 Branch Filtering: QA ONLY

Only commits to 'qa' branch trigger builds. Guard stage validates branch name early

🔢 Port Numbering Scheme

Format: 8 [ENV] [SERVICE]
Environment Codes:
  • 4 = Dev
  • 5 = QA (Active)
  • 6 = Production
Service Codes:
  • 00-99 = Different services
  • Range: 8500-8515
  • Example: 8500 = QA + Login

🔧 Troubleshooting Guide

❌ Build Failed: "Only QA branch allowed"

Cause:

Commit pushed to non-qa branch

Solution:

Merge or rebase to qa branch

❌ Container Failed to Start

Debugging Commands:
🔍 docker inspect <container>
🔍 docker logs <container>
🔍 netstat -tuln | grep 8500
🔍 docker network inspect ats-qa-network

❌ Service Communication Failure

Verification Steps:
✓ Ping service containers
✓ Check network connection
✓ Verify DNS resolution
