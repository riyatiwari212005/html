🎯 Problem Statement & Clinical Context

What is Diabetic Retinopathy?

Diabetic Retinopathy (DR) is damage to blood vessels in the retina caused by high blood sugar from diabetes. It is the leading cause of preventable blindness worldwide.

  • Affects ~1 in 3 people with diabetes
  • Often asymptomatic in early stages
  • Early detection enables timely treatment and intervention
  • Manual screening is time-consuming, subjective, and limited by specialist availability

Why AI-Based Automated Detection?

Current Clinical Challenge:
• Manual ophthalmologist screening: 10-15 minutes per patient
• Limited specialists: Few ophthalmologists in rural areas
• High cost: Professional examination ≈ $50-200 per patient
• Subjective: Inter-observer agreement varies (κ ≈ 0.75-0.85)
AI Solution Benefits:
✓ 24/7 automated screening at scale
✓ Cost-effective: Deploy on smartphone/tablet
✓ Consistent: Same algorithm, reproducible results
✓ Accessible: Reach remote/rural areas instantly

5 DR Severity Grades

Grade Classification Characteristics Clinical Action
0 No DR Normal retina, no lesions Routine follow-up
1 Mild Microaneurysms only Monitor annually
2 Moderate Microaneurysms + hemorrhages + hard exudates More frequent monitoring
3 Severe Multiple hemorrhages, cotton-wool spots, venous abnormalities Urgent evaluation needed
4 Proliferative Neovascularization, vitreous hemorrhage, risk of blindness IMMEDIATE treatment (laser/surgery)

🔧 Preprocessing Pipeline & Data Handling

Golden Rule: Preprocessing is 80% of the work.
Poor preprocessing → Even best models fail. Great preprocessing → Mediocre models work well.

What: Load images from disk, convert to RGB format

Why: CNNs expect 3-channel RGB. Red = vessels, Green = lesion contrast

DICOM/PNG/JPG → PIL Image.open() → .convert('RGB') → 3-channel tensor

What: Standardize all images to 224×224

Why:

  • Computational efficiency (84× smaller than 2048×2048)
  • ImageNet pretrained models use 224×224
  • Captures major lesions while staying practical
Trade-off:
Too small (<128px): Lose microaneurysm details ❌
224px: Optimal balance ✅
Too large (>512px): Memory intensive, minimal gain ⚠️

What: Apply channel-wise normalization

Statistics: Mean=[0.485, 0.456, 0.406], Std=[0.229, 0.224, 0.225]

Why:

  • Pretrained models optimized for these statistics
  • Zero mean & unit variance → stable training gradients
  • Partially compensates for illumination variation
normalized_pixel = (pixel - mean) / std

Augmentations Applied:

  • 🔄 Random Horizontal Flip (50%)
  • ↻ Random Rotation (±15°)
  • 🌈 Color Jitter (brightness ±20%, contrast ±20%)
  • 🔍 Random Zoom/Crop

Why:

  • Increases effective dataset size
  • Prevents overfitting
  • Simulates real-world variation (camera angles, lighting)
  • Improves generalization by 2-5% accuracy
⚠️ CRITICAL:
Augmentation ONLY on training data!
If applied before split → validation data leaks into training

What: Split data 70% train, 15% val, 15% test with stratification

Stratified means: Each class appears proportionally in all splits

Why essential for medical data:

  • No data leakage (no image in both train and test)
  • Prevents class imbalance bias (80% No-DR)
  • Fair validation that represents real performance
  • Prevents overfitting to test set
from sklearn.model_selection import train_test_split
train_df, temp_df = train_test_split(df, test_size=0.30, stratify=df['diagnosis'])
val_df, test_df = train_test_split(temp_df, test_size=0.50, stratify=temp_df['diagnosis'])

Problem: 80% No-DR, 2% Proliferative → Model biased

Solution: Inverse frequency class weighting

Formula: weight = total_samples / (num_classes × samples_in_class)

Effect: Rare classes weighted 5-10× higher in loss function

Clinical Reasoning:
False Negative (miss Severe) → Blindness ❌❌❌
False Positive (false alarm) → Doctor reviews ✅

Therefore: Weight rare classes to catch them!

What: Process 16-32 images at a time

Why:

  • Memory efficiency (8GB GPU can't load 3.6K images)
  • Stochasticity (random shuffling improves generalization)
  • Gradient stability (mini-batch averaging)
  • Reproducibility (fixed random seed)
train_loader = DataLoader(dataset, batch_size=16, shuffle=True)
val_loader = DataLoader(dataset, batch_size=16, shuffle=False)
test_loader = DataLoader(dataset, batch_size=16, shuffle=False)

Advanced Filtering & Enhancement Techniques for Retinal Images

Mathematical Theory

Gaussian Blur Formula:

G(x,y) = (1 / 2πσ²) × e^(-(x²+y²) / 2σ²)

G_blur = Original × Gaussian Kernel
where σ (sigma) = standard deviation
Purpose in Diabetic Retinopathy Detection
  • Noise Reduction: Smooths camera sensor noise, dust artifacts
  • Lesion Preservation: Large σ can blur out microaneurysms (use carefully!)
  • Background Uniformity: Makes optic disc stand out
  • Preprocessing for Derivatives: Essential before edge detection
Clinical Application
Recommended Parameters:
Kernel Size: 3×3 to 5×5 (preserve lesions)
σ (sigma): 0.5 to 1.5

Use Case: Initial noise reduction on RAW retinal images from imaging devices
Code Implementation
import cv2
import numpy as np

# Gaussian Blur
blurred = cv2.GaussianBlur(image, ksize=(5, 5), sigmaX=1.5, sigmaY=1.5)

# Effect: Smooth image, preserve edges moderately
⚠️ Caution: Excessive Gaussian blur can remove microaneurysms (tiny hemorrhages). Use σ ≤ 1.5 for medical images.

Mathematical Theory

Why Bilateral is Different from Gaussian:

Bilateral(x,y) = (1/W_p) × Σ I(ξ,η) × Gσ_s(||m-n||) × Gσ_r(|I(m,n)-I(ξ,η)|)

where:
Gσ_s = Spatial Gaussian (distance from center)
Gσ_r = Range Gaussian (intensity difference)
W_p = normalization factor
Key Advantage Over Gaussian Blur
Bilateral = "Smart Blur"

✓ Smooths within similar intensities (same tissue)
✓ PRESERVES sharp edges (lesion boundaries)
✓ Noise reduction without lesion loss
✗ Computationally expensive (O(w²) per pixel)
Clinical Application for DR Detection
  • Microaneurysm Preservation: Preserves tiny red dots while smoothing background
  • Hemorrhage Boundaries: Keeps sharp lesion edges
  • Hard Exudates: Maintains bright yellow spots without blurring edges
  • Vessel Clarity: Enhances vessel-background contrast
Recommended Parameters
d (diameter): 7-9 pixels
σ_color (range): 20-50
σ_space (spatial): 20-50

Typical Setting: d=7, σ_color=30, σ_space=30
Code Implementation
import cv2

# Bilateral Filter (edge-preserving)
bilateral = cv2.bilateralFilter(image, d=7, sigmaColor=30, sigmaSpace=30)

# Result: Smooth with sharp edges preserved
💡 Clinical Insight:
Bilateral filter is ideal for DR preprocessing because it removes fluorescence noise while keeping pathological lesion boundaries sharp for CNN feature extraction.

Mathematical Theory
Median(x,y) = median{I(ξ,η) : (ξ,η) ∈ neighborhood}

For a 3×3 window: Sort 9 pixel values, take middle value
Example: [10, 15, 20, 25, 30, 35, 40, 255, 0]
Median = 30 (eliminates outliers 255 and 0)
Why Median is Unique
Advantage: Non-linear & Robust to Outliers

✓ Excellent for impulse noise (sensor defects, dead pixels)
✓ Removes speckles without blurring
✓ Better than average for non-Gaussian noise
✓ O(n log n) complexity (manageable)
✗ Can create blocky artifacts if kernel too large
Clinical Application in Retinal Imaging
  • Sensor Artifacts: Removes dead pixels from fundus cameras
  • Dust Particles: Eliminates dust on optical elements
  • Vignetting Noise: Reduces peripheral artifacts
  • Preprocessing for OCT Images: Common first step for optical coherence tomography
Parameter Selection
Kernel Size: 3×3 or 5×5 (avoid >5×5 for medical images)

Medical Best Practice: Use 3×3 kernel
Larger kernels can obliterate microaneurysms
Code Implementation
import cv2

# Median Filter
median = cv2.medianBlur(image, ksize=5)

# ksize must be odd (3, 5, 7, etc.)
# Ideal for salt-and-pepper noise
When to Use Median:
If image has visible dead pixels/dots → Use Median 3×3
If image has Gaussian noise → Use Gaussian Blur
If you need both → Apply Median first, then Gaussian

Mathematical Theory
Global Histogram Equalization: CDF maps pixel values uniformly

Adaptive HE: Compute histogram locally in 8×8 blocks

Contrast Limiting: Clip histogram at threshold λ
λ = (M×N/L) × β
where M×N = block size, L = gray levels, β = clipping factor
Why CLAHE for Medical Imaging
Problem Solved: Over-enhancement in Standard HE

✗ Global HE: Makes dark optic disc region too bright
✗ Creates noise amplification in uniform regions

✓ CLAHE: Balances local and global contrast
✓ Clips histogram to prevent noise amplification
✓ Reveals microaneurysms and hard exudates
✓ Standard preprocessing in ophthalmology
Clinical Benefits for Diabetic Retinopathy
  • Microaneurysm Visibility: Makes tiny red spots 20% more visible
  • Hard Exudate Detection: Enhances yellow spots against red background
  • Vessel Segmentation: Improves blood vessel clarity
  • Optic Disc Definition: Brings disc boundaries into sharp relief
  • Cotton Wool Spots: Whitish lesions become prominent
Parameter Tuning
clipLimit: 2.0 - 4.0 (controls clipping intensity)
tileGridSize: 8×8 pixels (block size for adaptation)

Recommended for DR: clipLimit=2.5, tileGridSize=(8,8)
Code Implementation
import cv2
import numpy as np

# Convert to LAB (better than RGB for enhancement)
lab = cv2.cvtColor(image, cv2.COLOR_BGR2LAB)
l, a, b = cv2.split(lab)

# Apply CLAHE to L channel only
clahe = cv2.createCLAHE(clipLimit=2.5, tileGridSize=(8,8))
l_enhanced = clahe.apply(l)

# Merge and convert back
lab_enhanced = cv2.merge([l_enhanced, a, b])
result = cv2.cvtColor(lab_enhanced, cv2.COLOR_LAB2BGR)
💡 Pro Tip: Apply CLAHE to LAB color space L-channel only, not RGB. This preserves color information while enhancing contrast (LAB L = Lightness).

Thresholding Concepts
Simple Threshold: Output = 255 if Input > T else 0

Otsu's Method: Finds threshold T that maximizes between-class variance
T_optimal = argmax(σ_b²) = argmax(w₀ × w₁ × (μ₀ - μ₁)²)

Adaptive: T(x,y) = mean(neighborhood(x,y)) - C
where C = bias constant
Three Thresholding Approaches
Method How It Works Best For Limitation
Simple Fixed threshold T High contrast images Fails with uneven lighting
Otsu Auto-threshold, maximizes variance Bimodal histograms Assumes two distinct classes
Adaptive Local threshold per pixel neighborhood Uneven illumination (MEDICAL STANDARD) Slower computation
Clinical Application: Blood Vessel Segmentation
Use Case: Extract Blood Vessels for Analysis

1. Apply Gaussian to smooth
2. Extract green channel (vessels are dark green in retinal images)
3. Apply Adaptive Threshold → Binary mask of vessels
4. Use for hemorrhage detection (should be within vessels)
Code Implementation
import cv2
import numpy as np

# Convert to grayscale
gray = cv2.cvtColor(image, cv2.COLOR_BGR2GRAY)

# Method 1: Otsu Thresholding
_, thresh_otsu = cv2.threshold(gray, 0, 255, cv2.THRESH_BINARY + cv2.THRESH_OTSU)

# Method 2: Adaptive Thresholding (MEDICAL STANDARD)
thresh_adaptive = cv2.adaptiveThreshold(gray, 255, cv2.ADAPTIVE_THRESH_GAUSSIAN_C,
    cv2.THRESH_BINARY, blockSize=11, C=2)

# Method 3: Manual threshold
_, thresh_manual = cv2.threshold(gray, 127, 255, cv2.THRESH_BINARY)
Which Method to Use?

Uniform lighting + 2 classes → Otsu
Uneven illumination (real cameras) → Adaptive ✓ BEST
Specific pathology → Manual tuning

Mathematical Theory
Sobel X-Gradient: Gx = [[-1,0,1], [-2,0,2], [-1,0,1]] ⊗ Image

Gradient Magnitude: G = √(Gx² + Gy²)
Gradient Direction: θ = atan2(Gy, Gx)

Morphological Operations Theory
Erosion: Output = min of all pixels under kernel
E(x,y) = min{I(x+i, y+j) : (i,j) ∈ Kernel}

Dilation: Output = max of all pixels under kernel
D(x,y) = max{I(x+i, y+j) : (i,j) ∈ Kernel}

Opening = Erosion followed by Dilation (remove small objects)
Closing = Dilation followed by Erosion (fill small holes)

Gradient = Dilation - Erosion (edge detection variant)
Four Main Operations & Their Effects
Operation Formula Effect Medical Use
Erosion Shrink white regions Removes small objects, thins lines Remove noise, isolate lesions
Dilation Expand white regions Enlarges objects, fills gaps Connect broken vessels, fill holes
Opening Erosion → Dilation Remove small noise, preserve large objects MOST USED - Clean up false positive noise
Closing Dilation → Erosion Fill holes, connect nearby objects Merge fragmented hemorrhages
Clinical Application in Diabetic Retinopathy
Lesion Refinement Pipeline:

1. Threshold → Binary lesion map (contains noise)
2. Opening (3×3) → Remove false positives (dust, noise)
3. Closing (5×5) → Merge fragmented hemorrhages
4. Morphological Gradient → Find lesion boundaries
5. Label connected components → Individual lesions
Kernel (Structuring Element) Selection
  • 3×3 Rectangular: Fast, for small noise removal
  • 5×5 Elliptical: Better for circular lesions (exudates)
  • 7×7 Cross: For vessel connectivity
  • Custom: Design based on lesion morphology
Detailed Effects & Parameters
Microaneurysm (1-3 px): 3×3 kernel erosion removes it
Small hemorrhage (5-15 px): 3×3 opening refines, 5×5 closing merges fragments
Large hemorrhage (>20 px): Preserve with 5×5 or larger kernel
Hard exudate clusters: 5×5 closing effective
Code Implementation
import cv2
import numpy as np

# Define structuring element (kernel)
kernel_3x3 = cv2.getStructuringElement(cv2.MORPH_ELLIPSE, (3, 3))
kernel_5x5 = cv2.getStructuringElement(cv2.MORPH_ELLIPSE, (5, 5))

# Erosion - removes small objects
eroded = cv2.erode(binary_image, kernel_3x3, iterations=1)

# Dilation - fills holes, connects nearby objects
dilated = cv2.dilate(binary_image, kernel_3x3, iterations=1)

# Opening = Erosion + Dilation (remove noise)
opened = cv2.morphologyEx(binary_image, cv2.MORPH_OPEN, kernel_3x3)

# Closing = Dilation + Erosion (fill holes)
closed = cv2.morphologyEx(binary_image, cv2.MORPH_CLOSE, kernel_5x5)

# Morphological Gradient = Dilation - Erosion (edges)
gradient = cv2.morphologyEx(binary_image, cv2.MORPH_GRADIENT, kernel_3x3)
Advanced: Multi-Level Morphology
# Remove noise, then refine lesions
step1_denoise = cv2.morphologyEx(binary, cv2.MORPH_OPEN, cv2.getStructuringElement(cv2.MORPH_ELLIPSE, (3,3)))
step2_merge = cv2.morphologyEx(step1_denoise, cv2.MORPH_CLOSE, cv2.getStructuringElement(cv2.MORPH_ELLIPSE, (5,5)))
step3_edges = cv2.morphologyEx(step2_merge, cv2.MORPH_GRADIENT, cv2.getStructuringElement(cv2.MORPH_ELLIPSE, (3,3)))

# Result: Clean, connected lesions with defined boundaries
💡 Pro Strategy for DR:
After thresholding lesions, always apply Opening first (removes noise), then Closing (connects fragments). This 2-step approach handles most artifacts while preserving pathology.

📈 Comprehensive Evaluation Metrics

⚠️ Critical Insight:
Accuracy ALONE is misleading for medical AI!
Model predicting "No-DR" for everything → 80% accuracy but catches 0% of disease

Why Multiple Metrics Matter

Metric What It Measures Clinical Significance When to Use
Accuracy Overall correctness Easy to communicate Initial sanity check
Precision False alarm rate Of positive predictions, how many correct? Avoid unnecessary follow-ups
Recall (Sensitivity) Miss rate Of actual disease, what % caught? PRIMARY METRIC for DR (must be ≥90%)
Specificity True negative rate Of healthy people, how many correctly ruled out? Balance with sensitivity
F1-Score Harmonic mean P&R Balanced metric When P and R equally important
ROC-AUC Ranking ability Most important for medical AI (FDA standard) GOLD STANDARD (target ≥0.90)
Quadratic Kappa Expert agreement Accounts for ordinal nature (Mild vs Severe) Grading tasks (target ≥0.85)
Confusion Matrix All prediction types Shows confusion patterns Debugging & error analysis

Understanding ROC-AUC (Most Important)

Formula: Area under curve of TPR vs FPR
Interpretation: Probability model ranks positive sample higher than negative
Benchmark: 0.5 = random, 0.9+ = excellent
Why for DR: Threshold-independent, handles imbalance, FDA requirement

Understanding Precision vs Recall Trade-off

Precision (Low False Alarms):
Of our positive predictions, how many actually DR?
When to prioritize: Administrative (reduce unnecessary referrals)

Recall (Low Misses):
Of actual DR cases, how many did we catch?
When to prioritize: Clinical (MUST catch disease)
DR Detection Priority:
False Negative (miss Severe) = Patient blindness = CATASTROPHIC ❌❌❌
False Positive (false alarm) = Doctor reviews = ACCEPTABLE ✅

Decision: Optimize for RECALL ≥ 90%

⚠️ Current Limitations & Where We Are Tight

Data Challenges

  • Lack of diverse ethnic data: Most datasets from USA/Europe/India. African/Asian/Latin American populations underrepresented
  • Annotation is expensive & slow: Expert ophthalmologist annotation = $8-16 per image, takes weeks for 1000 images
  • Severe DR samples rare: Only 2-3% of real-world data, hard to train models with high recall for critical cases
  • Camera-specific artifacts: Zeiss vs Topcon vs other brands have different illumination, contrast profiles

Clinical Deployment Issues

  • Domain shift between hospitals: Model trained on EyePACS → deployed at Apollo Sage Hospital → 5-10% accuracy drop due to different cameras, demographics, lighting
  • Trust & explainability requirements: Doctors refuse to use "black box" AI. Need attention maps, segmentation masks, or other explanations
  • Regulatory approvals: FDA 510(k) approval requires 1-2 years, multi-center validation on 1000+ subjects
  • Data privacy regulations: GDPR, HIPAA require explicit consent, secure storage, right-to-deletion protocols

Technical Limitations

  • Models fail on poor-quality images: Motion blur, cataracts, floaters → 10-15% accuracy degradation
  • Difficulty distinguishing adjacent grades: 85% of classification errors are Mild↔Moderate or Moderate↔Severe (ordinal adjacency)
  • Vision Transformers need large datasets: ViT requires 10k+ images for stable fine-tuning. With <5k images, overfits to patch tokens
  • Inference latency: ViT (250ms) much slower than MobileNet (50ms) - not suitable for real-time screening booths

Ethical & Legal Issues

  • Bias in datasets: EyePACS ~88k images mostly from USA diabetics (predominantly African American, Hispanic). Different ethnic backgrounds have different vessel patterns
  • Liability in misdiagnosis: If model misses Severe DR → patient goes blind → hospital legally responsible. Insurance may not cover AI-caused errors
  • De-anonymization risk: Fundus images have unique vessel patterns → can identify individual even with name removed

🚀 Future Research Directions

Multimodal Learning

Vision + Medical Data:
Combine fundus photos + OCT scans + patient EHR (diabetes duration, HbA1c, blood pressure) → Unified model
Expected improvement: 98%+ accuracy vs 97% with fundus alone
Clinical value: Predict progression 6-12 months earlier

Longitudinal Analysis (Temporal)

Track same patient over years:

• Future: Patient history (visit 1: No-DR, visit 2: Mild, visit 3: Moderate) → Predict Grade at visit 4
• Use LSTM/Transformer temporal models to learn progression patterns
• Clinical impact: Intervene before patient worsens

Foundation Models (Self-Supervised)

Problem: Current models require 1000+ labeled images per hospital
Solution: Pretrain on 100k+ unlabeled fundus images using contrastive learning (SimCLR, MoCo)
Benefit: Transfer to new hospitals with only 100 labeled images needed (not 1000+)
Timeline: 2-3 years for mature implementation

Federated Learning (Privacy-Preserving)

Traditional: Send all images to cloud → Model improves
Problem: GDPR violation, privacy risk, slow upload from rural hospitals
Federated approach:
1. Each hospital trains model locally on its patients
2. Only send model weight updates (not raw images) to central server
3. Aggregate updates (Federated Averaging) → global model improves
4. Each hospital downloads improved global model
Advantage: Data never leaves hospital, privacy protected, continuous learning

Real-Time Video Analysis

Current: Single static fundus photo → Classification
Future: Continuous video stream during retinal examination → Process 30 fps
Benefit: More spatial information, temporal consistency, detect rare lesions across multiple frames
Challenge: Requires optical flow networks to align frames

Explainable AI (XAI) - Enhanced

Current explanations: Attention maps (which patches matter)

Future XAI methods:
  • Grad-CAM: Gradient-weighted class activation mapping - shows which pixels influenced prediction
  • LIME: Local Interpretable Model-agnostic Explanations - approximate model locally with interpretable model
  • Shapley Values: Game theory approach - quantify contribution of each image region to prediction
  • Concept Activation Vectors (CAVs): "This image has prominent hemorrhages" (human-interpretable concepts, not pixels)
  • Counterfactual Explanations: "If hemorrhage was 5 pixels smaller, grade would be Moderate not Severe"

📊 Clinical Decision Thresholds & Confidence

How to Set Thresholds for Real-World Deployment

Problem: Model outputs softmax probabilities [0, 1] per class. What threshold for deciding "DR present"?

Example:
Image A: P(No-DR)=0.51, P(Mild)=0.30, P(Moderate)=0.19
→ Default: Predict "No-DR" (highest probability)
→ But only 51% confident!

Clinical question: Is 51% confidence enough to tell patient "you're healthy"?

Three-Tier Clinical Triage System

Confidence Level Threshold Action Clinical Rationale
High Confidence > 0.95 ✅ Auto-approve & process (document result) Model very sure. Surgeon rarely needed to review.
Medium Confidence 0.80-0.95 ⚠️ Flag for human review (doctor validates) Uncertain cases. Domain expert must verify before release.
Low Confidence < 0.80 ❌ Refer to ophthalmologist (always review) Model unclear. Specialist diagnosis required.

Sensitivity vs Specificity Trade-off

Clinical Priority: Catch disease first, false alarms second

Scenario 1: Conservative (High Sensitivity)
• Threshold = 0.4 (aggressive)
• Sensitivity: 95% (catch 95% of DR cases)
• Specificity: 70% (but 30% false alarms on healthy patients)
• Cost: Extra specialist reviews but saves sight

Scenario 2: Balanced
• Threshold = 0.5 (default)
• Sensitivity: 90%, Specificity: 88%
• Reasonable for most hospitals

Scenario 3: Aggressive (High Specificity)
• Threshold = 0.7 (conservative)
• Sensitivity: 70% (miss 30% of DR cases!)
• Specificity: 95% (few false alarms)
• ❌ NOT RECOMMENDED: Too many misses = preventable blindness
Clinical Recommendation:
Always optimize for SENSITIVITY ≥ 90% in clinical deployment
Missing DR = patient blindness
False alarm = doctor reviews (low cost)

🔬 Advanced Preprocessing Techniques

Contrast Enhancement (CLAHE)

What: Contrast Limited Adaptive Histogram Equalization

How: Divide image into 8×8 tile regions. Apply histogram equalization locally.

Benefits:
✓ Restores vessel visibility in poorly lit images
✓ Increases contrast of hard exudates (appear brighter, more distinct)
✓ Reduces impact of illumination variation across retina

Risk: Over-enhancement amplifies image noise, makes artifacts look like lesions

Best Practice: CLAHE with clip_limit=2.0, tile_size=8×8 (OpenCV defaults)

Gaussian Filtering for Noise Removal

What: Low-pass filter to remove high-frequency noise

Sigma parameter: Gaussian standard deviation
• Sigma=0.5: Minimal smoothing, preserves all details
• Sigma=1.5: Good balance (recommended)
• Sigma=3.0: Heavy smoothing, loses fine details

Challenge: Must be light enough to preserve microaneurysms (15-30 pixels) but heavy enough to remove camera noise

Solution: Sigma=1.5 → microaneurysms (15px) preserved, noise (1-2px) removed

Illumination Normalization

Problem: Different cameras, lighting conditions create brightness variation across image

Solution approaches:
1. Global normalization: Divide each pixel by average brightness
2. Local normalization: Divide by local mean (background removal)
3. Gamma correction: Adjust pixel^gamma to match reference brightness

Applied in preprocessing: After resize, before augmentation, to stabilize lighting

Focal Loss for Class Imbalance

Problem: Class weighting helps but doesn't focus enough on hard negatives

Example:
• Easy negative: No-DR image clearly has no lesions (model predicts correctly with 0.98 confidence)
• Hard negative: No-DR image with small vessels mimicking microaneurysms (model confused, 0.55 confidence)

Standard loss: Penalizes all errors equally
Focal loss: Down-weights easy examples, focuses on hard ones

Formula: FL = -α(1-p)^γ * log(p)
• γ=2 focuses on hard examples
• γ=0 reduces to standard cross-entropy

Result: +2-3% improvement on hard cases
